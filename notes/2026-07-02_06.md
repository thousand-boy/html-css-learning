# 今日学んだこと
### 2026-07-02〜06｜第5章 2カラムのWebサイト制作 5-1〜5-4前半（2カラムの概念／制作の流れ／ページ見出し作成・`news.html` の準備／メインエリア）

**何をしたか**
- **5-1 2カラムのレイアウトとは**：「カラム」＝縦の列のこと。垂直方向にコンテンツを区切って組むレイアウトを「カラムレイアウト」と呼ぶと学びました。この章ではよくある店舗のお知らせを載せる **「NEWS」ページ** を作成します。2カラムは「2列に並んだレイアウト」で、コンテンツ量の多いニュースサイト・ブログの見せ方に向くこと、構成要素として **ページ上部の画像 `<div>`／見出し `<h2>`／メインエリア `<article>`／サイドバー `<aside>`** があることを確認しました。
- **5-1（続き）モバイル配置・幅比率**：モバイルなど画面が小さい時は要素が縦に並ぶこと、デスクトップではメインエリアとサイドバーを **「2:1」や「3:1」** の比率で組むのが一般的で、半分に割るものは **「スプリットスクリーン」** と呼ぶことを学びました。
- **5-2 2カラムページの制作の流れ**：ホームと同じく **モバイルサイズから** 作成する方針を確認。流れは ①ページ全体の見出しを作成 → ②メインエリアの作成 → ③サイドバーの作成（カテゴリー一覧・アーカイブ）→ ④レスポンシブ対応（デスクトップで2カラム化）。POINT として **カラム間の余白** をしっかり入れることを学びました。
- **5-3 ページ全体の見出しを作成しよう（ファイルの準備）**：ロゴ・ナビ・フッターは前章の `index.html` と共通なので、VS Code 左側の `index.html` を右クリック →［コピー］→ 枠内で右クリック →［貼り付け］で複製し、**`news.html`** とリネームして保存しました。
- **5-3（続き）HTML を編集**：`<head>` 内の `<title>` を **`WCB Cafe - NEWS`** に変更し、`description` も調整（c5-03-1）。ホーム固有の **`<section class="about">` を削除**（c5-03-2）。
- **5-3（続き）クラス名・見出しを変更**：ヘッダーを囲む `<div>` のクラスを **`cover cover-home` → `sub-cover cover-news`** に変更（c5-03-3）。見出しの `<h2 class="page-title font-english">` のテキストを **「News」** に変更（c5-03-4）。
- **5-3（続き）CSS で装飾**：前章の `style.css` に **`.sub-cover`（高さ・余白）** と **`.cover-news`（News 用背景画像）** を追記（c5-03-5）。
- **5-4 メインエリアを作ろう**：記事上部のタイトル・日付・カテゴリーを HTML で作成。全体を **`<main class="wrapper">`** で囲み、`<article class="post">` → `<header>` の中に見出し・カテゴリー（`<ul>`）・日付（`<time datetime>`）を配置しました（c5-04-1）。

**学んだ用語**

| 用語 | やさしい説明 |
|---|---|
| カラム／カラムレイアウト | 「カラム」は縦の列。垂直方向にコンテンツを区切って組むレイアウト |
| 2カラムレイアウト | 2列に並んだレイアウト。コンテンツ量の多いニュース・ブログの見せ方に向く |
| ページ上部の画像 `<div>` | ページ最上部に置く大きな画像（カバー）の領域 |
| メインエリア `<article>` | 記事本体が入る主要コンテンツ領域 |
| サイドバー `<aside>` | カテゴリー一覧やアーカイブなど、補足情報を置く領域 |
| カラムの幅比率 | メインエリア:サイドバー＝「2:1」や「3:1」が一般的 |
| スプリットスクリーン | 画面を半分で割る2カラムレイアウトの呼び方 |
| ファイル複製（`news.html`） | 共通部分が多いページは `index.html` を複製して編集するとミスが少ない |
| `sub-cover` / `cover-news` | ホームの `cover`／`cover-home` を News 用に置き換えたクラス名（広げ方と画像を分けて管理） |
| `<main>` | ページの主要コンテンツを表すタグ。`wrapper` クラスで左右に余白 |
| `wrapper`（前章で定義） | 最大幅＋左右余白＋中央寄せの共通クラス |
| `<article class="post">` | 独立した1つの記事のまとまり |
| `post-cat`（`<ul>`） | カテゴリーは複数付く前提で箇条書きリストにする |
| `<time datetime="…">` | 日時を表すタグ。`datetime` は「年-月-日」形式。検索エンジンに拾われ検索結果に反映される |

**できるようになったこと**：2カラムレイアウトの概念（メイン `<article>` とサイドバー `<aside>`、幅比率 2:1／3:1、スプリットスクリーン）の理解。共通部分の多いページを `index.html` の複製で用意し、`news.html` として `title`／`description` を編集し不要な `about` を削除する手順。ヘッダーを囲む `<div>` のクラスを `sub-cover cover-news` に変更し、見出しを「News」に差し替え。`style.css` への `.sub-cover`・`.cover-news` 追記。`<main class="wrapper">`＋`<article class="post">` によるメインエリアの作成（カテゴリーの `<ul>`、`<time datetime>` を含む記事情報部分）。

**つまずき・気づき**
- 共通部分が多いページは、ゼロから書かず **既存 HTML（`index.html`）を複製 → 編集** するとミスが減る。ホーム固有の `about` セクションは News では不要なので削除する。
- クラスは **「広げ方（`.cover` → `.sub-cover`）」と「ページ固有の画像（`.cover-home` → `.cover-news`）」を分けて** 管理する設計。下層ページで共通の `.sub-cover` を使い回せる。
- `<time>` の `datetime` は **「年-月-日」形式**。表示テキスト（例：2024年9月28日）と機械可読値（`2024-09-28`）を分けられ、検索エンジンに拾われて検索結果に反映される（SEO に有効）。
- カテゴリーは複数付く前提で **`<ul>` の箇条書きリスト** にしておく。
- 2カラムは **幅比率（2:1／3:1）とカラム間の余白** が「窮屈にしない」ためのポイント。まずモバイル、次にデスクトップ（レスポンシブ）で2カラム化する流れ。

### 2026-07-02〜06（続き）｜第5章 5-4後半〜5-7（記事の装飾／サイドバー／レスポンシブで2カラム化／3カラムのカスタマイズ）＋第6章 6-1 冒頭

**何をしたか**
- **5-4 タイトルと日付を装飾しよう**：記事全体を囲む `.post` に下余白、`.post-title` の太さ調整、`.post-date` の文字サイズを指定（c5-04-2）。
- **5-4 カテゴリーと日付を横並びに**：`.post-info` に `display: flex` と `gap` を指定して、カテゴリーと日付を横並びに（c5-04-3）。
- **5-4 カテゴリーの装飾**：`.post-cat` を `display: flex`＋`list-style: none` で横並びにし、`<a>` に背景色・角丸・余白を付けてボタン風に（c5-04-4）。COLUMN で **代表的なリセットCSS（`normalize.css`／`ress.css`）** を確認。
- **5-4 画像と本文を書こう**：`</header>` の下に `<img class="post-thumbnail">` と本文の `<p>` を追加し、CSS で画像・段落の下余白を指定して **記事部分が完成**（c5-04-5）。
- **5-5 サイドバーを作ろう**：`<article>` の下に `<aside class="sidebar">` を追加。`Category`・`Archives` の見出し（`heading-medium`）と、`<ul class="side-menu">` のリンクリストを作成（c5-05-1）。見出しは `.page-title, .heading-large, .heading-medium` を **カンマで共通化**（c5-05-2）。`.side-menu` の丸マークを消し、`<a>` を `display: block` にしてリスト幅いっぱいをクリック可能にし、下線・余白・ホバー色を指定（c5-05-3）。
- **5-6 レスポンシブに対応させよう**：メディアクエリー（`@media (min-width: 800px)`）内で見出し・記事タイトルを拡大、ヘッダーを Flexbox 化、`.sub-cover` を低く控えめに、背景画像を **デスクトップ用（`-l.webp`）** に差し替え。`<main>` に **`news-contents` クラス** を追加し、`display: flex`＋`justify-content: space-between` と **`%` 幅指定** で **メイン70%／サイド22%（間8%）の2カラム** に（c5-06-4）。
- **5-7 2カラムページのカスタマイズ例（3カラム化）**：`news-contents` 内に `<aside class="ad">`＋バナー画像を追加して3カラム化（c5-07-1）。メインを70%→60%に調整し（c5-07-2）、**`order` プロパティ** でバナー→メイン→サイドの表示順に並べ替え（c5-07-3）。COLUMN で **重要コンテンツは HTML 上部に書く（SEO配慮）** ことを確認。
- **第6章 6-1（冒頭のみ）**：次章「タイル型のWebサイト」の導入。**タイル型レイアウト**＝画像やテキストを敷き詰めて一度に見せる形で、ショッピングサイトの商品一覧などでよく使われる、というところまで確認。

**学んだ用語**

| 用語 | やさしい説明 |
|---|---|
| `.post-info` に `display: flex` | カテゴリーと日付を横並びにする起点 |
| `gap` | フレックスアイテム同士の間隔 |
| `.post-cat`（`display: flex`＋`list-style: none`） | カテゴリー項目を横並びにし、リストの丸マークを消す |
| `.post-cat a`（背景色・`border-radius`・余白） | リンクをボタン風に装飾（ホバーで背景色変化） |
| `.post-thumbnail` | 記事内のサムネイル画像用クラス（下余白を指定） |
| `<aside class="sidebar">` | サイドバー本体（Category／Archives を格納） |
| `.heading-medium` | 中サイズ見出しの共通クラス（`page-title`・`heading-large` とカンマで共通化） |
| `.side-menu`（`list-style: none`） | サイドバーのリンクリスト。丸マークを消し中央揃え |
| `display: block`（`.side-menu a`） | リンクをブロック化し、リスト項目の横幅いっぱいをクリック可能にする |
| `news-contents`（`<main>` に付与） | メインとサイドを囲む親。`display: flex` で横並びにするためのクラス |
| `justify-content: space-between` | 子要素を両端に揃える→間に自動で余白が生まれる |
| 幅の `%` 指定 | 画面幅が変わっても伸縮する幅指定（例：メイン70%／サイド22%／余白8%） |
| `<aside class="ad">` | 3カラム目（縦長バナー画像）の領域 |
| `order` プロパティ | Flexbox 内で表示順を数値で指定。HTML の記述順とは別に見た目の順序を変える |
| リセットCSS（`normalize.css`／`ress.css`） | ブラウザー既定のスタイル差をならすCSS（COLUMN） |
| タイル型レイアウト（6-1） | 画像・テキストを敷き詰めて一度に見せるレイアウト（次章） |

**できるようになったこと**：記事の装飾（`.post-info`／`.post-cat` の `display: flex` 横並び、カテゴリーのボタン風装飾、`.post-thumbnail` と本文の追加）。サイドバー（`<aside class="sidebar">`）の作成と装飾（`side-menu` の丸マーク非表示・`display: block`・`border-bottom`）。`.heading-medium` のカンマ共通化。メディアクエリーでのデスクトップ2カラム化（`news-contents` に `display: flex`、メイン70%／サイド22%、背景画像を `-l.webp` に差し替え、`.sub-cover` を控えめに）。`<aside class="ad">` 追加と `order` プロパティによる3カラム化と順序変更。「重要コンテンツは HTML 上部」という原則の理解。

**つまずき・気づき**
- 横並びは **親要素に `display: flex`**。子同士の間隔は `gap`、両端揃え＋自動余白は `justify-content: space-between`。
- カラム幅は **`%` 指定** にすると画面幅が変わっても伸縮する。2カラムは 70%＋22%＋8%(余白)＝100% の配分。
- リンクは `display: block` にすると、テキスト部分だけでなく **リスト項目の横幅いっぱい** がクリック可能になる。
- **`order` は `display: flex`（や grid）内でのみ有効**。HTML の記述順を変えずに見た目の順序だけ変えられる。
- **重要なコンテンツ（記事）は HTML のなるべく上部** に書くのがベスト（読み込み順・SEO）。見た目の並び替えは CSS の `order` で行う。
- 見出しは `page-title`／`heading-large`／`heading-medium` を **カンマで共通化** し、デスクトップではメディアクエリー内で個別にサイズ変更する。

**学んだ用語**

| 用語 | やさしい説明 |
|---|---|
| CSSグリッド（`display: grid`） | 要素を格子状（タイル型）に並べるレイアウト手法 |
| `grid-template-columns` | 列の数と幅を指定。`repeat(2, 1fr)` は「同じ幅の列を2つ」 |
| `1fr` | グリッドで使う伸縮単位（余ったスペースの割合）。画面幅に応じて伸縮 |
| `repeat(回数, 幅)` | 同じ列指定をまとめて書く関数。`repeat(3, 1fr)`＝1fr列を3つ |
| `gap`（例 `2rem 1.5rem`） | グリッド項目同士の間隔（縦 横 の順） |
| `aspect-ratio: 1/1` | 表示する比率を指定（1/1＝正方形） |
| `object-fit: cover` | 指定サイズからはみ出す部分を自動トリミング（画像の歪み防止） |
| `.item` / `.grid` | タイル1枚（画像＋テキスト）と、それらを囲む親のクラス |
| `item-big` | 特に目立たせたい1枚を大きく表示するための追加クラス |
| `grid-column` / `grid-row` | グリッドライン番号で項目の横／縦の範囲を指定（`1/3`＝1〜3番ライン＝2マス分） |
| グリッドライン | 列・行を区切る線。左端・上端を1番目と数える |

**つまずき・気づき**
- **`grid-template-columns: repeat(2, 1fr)`** のように `1fr` を使うと、画面幅が変わっても列が伸縮する。列数を変える（`repeat(3, 1fr)`）だけでレスポンシブ対応できるのがグリッドの利点。
- 画像は **`aspect-ratio: 1/1`** で比率指定。元画像が正方形でないと歪むため、**`object-fit: cover`** をセットで書いてはみ出し部分をトリミングする。
- `gap` は **「縦 横」の順** で2値指定できる（例 `gap: 2rem 1.5rem`）。
- **`grid-column`／`grid-row` はグリッドライン番号** で範囲を指定する。左端・上端が「1」で、`1/3` は「1番ラインから3番ラインまで＝2マス分」を意味する。
- グリッド項目の並べ替え・拡大は、HTML の順序を変えずに CSS 側（`grid-column`／`grid-row`、前章の `order`）で制御できる。
