**何をしたか**
- 概念中心だった第1章を終え、いよいよ **実際にHTMLを書く** 第2章（2-1〜2-9）に入りました。
- VS Code で `index.html` を作成・保存し、ブラウザーで表示確認する **一連の流れ** を習得しました。
- `<!doctype html>`〜`<body>` という HTMLの **骨組み** と、文字コードや言語の指定（`meta charset` / `lang`）を理解しました。

- 見出し（`h1`〜`h6`）・段落（`p`）・画像（`img`）・リンク（`a`）・リスト（`ul`/`ol`/`li`）の **基本タグ** を一通り書けるようにしました。
- 画像やリンクで必要になる **ファイルパス（相対パス／絶対パス）** の指定方法を学びました。

**学んだ用語**
| 用語 | やさしい説明 |
|---|---|
| HTML | Hyper Text Markup Language の略。Webページの土台を作る言語 |
| タグ | `<` と `>` で囲んだ文字列。文章に「意味」を与える目印 |
| 要素 | 開始タグ＋コンテンツ＋終了タグ をまとめた1つのかたまり |
| 開始タグ／終了タグ | `<p>` が開始、`</p>` が終了。`/` 付きが終了側 |
| 属性 | タグに付ける付加情報（例：`href`, `src`, `alt`, `lang`） |
| 値 | 属性の中身。`"..."`（ダブルクォーテーション）で囲む |
| 入れ子（ネスト） | タグの中にタグを入れること。先に開けたタグを後に閉じる |
| Doctype宣言 | `<!doctype html>`。そのページがどのHTML仕様かを示す |
| HTML Living Standard | 現在主流のHTML仕様。WHATWGが継続更新する「生きている標準」 |
| lang属性 | `<html lang="ja">` のように文書の言語を指定（ja＝日本語） |
| head | ページ情報を書く領域。画面には表示されない |
| meta要素 | ページのメタ情報（文字コードや説明文など）を示すタグ |
| 文字コード／UTF-8 | 文字をデータで表す方式。UTF-8 指定で文字化けを防ぐ |
| title | タブ・ブックマーク・検索結果に出るページタイトル |
| description | ページの説明文。検索結果でタイトルと共に表示される |
| body | HTMLの本体。ここに書いた内容が実際に表示される |
| 拡張子 | ファイル名の `.` より右の部分（`.html` 等）。ファイルの種類を示す |
| 相対パス | 呼び出し元ファイルから見た、対象ファイルの位置の書き方 |
| 絶対パス | `http://` `https://` から始まる完全なURLでの指定 |
| src属性 | `<img>` で表示する画像の場所を指定する属性 |
| alt属性 | 画像が表示できない時の代替テキスト。画像の意味を伝える |
| href属性 | `<a>` のリンク先（URLやパス）を指定する属性 |
| mailto | `href="mailto:..."` でメール作成画面を開くリンク指定 |
| target属性 | `target="_blank"` でリンクを別タブに開く |
| h1〜h6 | 見出しタグ（heading）。h1が最大、数字が大きいほど小さい |
| p | 段落タグ（Paragraph）。文章のまとまりを表す |
| img | 画像タグ。終了タグがなく単独で使う |
| a | リンクを作るタグ（anchor） |
| ul／li | 順序なし箇条書きリスト（Unordered List）と項目（List Item）。黒丸 |
| ol／li | 順序あり番号付きリスト（Ordered List）と項目。連番 |

**学んだ主な構文**
```html
<!doctype html>                         <!-- Doctype宣言 -->
<html lang="ja">                        <!-- 日本語文書 -->
  <head>                                <!-- ページ情報（非表示） -->
    <meta charset="UTF-8">              <!-- 文字コード -->
    <title>猫の実態</title>             <!-- タイトル -->
    <meta name="description" content="猫の好きなもの、日々の生活をご紹介"> <!-- 説明文 -->
  </head>
  <body>                                <!-- 本体（表示される） -->
    <h1>猫の一日</h1>                    <!-- 見出し（h1＝最大） -->
    <p>ひたすら寝ています。</p>          <!-- 段落 -->
  </body>
</html>
```
```html
<!-- 見出しは h1 から順に使う（見た目目的で飛ばさない） -->
<h1>1番大きな見出し</h1>
<h2>2番目の見出し</h2>
<h3>3番目の見出し</h3>
```
```html
<!-- 画像：終了タグなし。src=画像の場所, alt=代替テキスト -->
<img src="sakura.jpg" alt="美しい桜">             <!-- 同じフォルダ -->
<img src="images/sakura.jpg" alt="美しい桜">      <!-- 1つ下のフォルダ -->
<img src="../images/sakura.jpg" alt="美しい桜">   <!-- 1つ上の階層 -->
```
```html
<!-- リンク：href=リンク先。別サイトは http(s):// を必ず付ける -->
今すぐ<a href="https://google.co.jp">Google</a>で検索！
<!-- 画像にリンク：<a>で<img>を囲む -->
<a href="https://google.co.jp"><img src="sakura.jpg" alt="美しい桜"></a>
<!-- メール宛てリンク -->
<a href="mailto:info@example.com">お問い合わせ</a>
```
```html
<!-- 箇条書きリスト（黒丸） -->
<ul>
  <li>卵（3個）</li>
  <li>しょうゆ（大さじ1）</li>
</ul>
<!-- 番号付きリスト（連番） -->
<ol>
  <li>卵を割りほぐし、調味料を加える</li>
  <li>卵焼き器に卵液を1/4量入れて広げる</li>
</ol>
```

**できるようになったこと**: HTMLの骨組み（`<!doctype html>`〜`<body>`）を自力で書き、見出し・段落・リストで文章構造を組み立てられる。`<img>` の `src`／`alt` と相対パス（同階層・下層・`../`）を使い分けて画像を表示でき、`<a href>` でページ間・外部サイト・メール宛てのリンクを作れる。タグの入れ子と `属性="値"` の書き方も正しく扱える。

**つまずき・気づき**
- `<img>` に **終了タグは不要**（`</img>` と書かない）。単独で使う。
- 入れ子は **交差させない**。`<p><strong>…</strong></p>` はOK、`<p><strong>…</p></strong>` はNG。
- 属性の値は必ず **ダブルクォーテーション** で囲む（`href="..."`）。
- 外部サイトへのリンクは `http://`／`https://` を **忘れない**。`google.co.jp` だけでは移動しない。
- 見出しは **順番厳守**。見た目の大きさ目的で `<h1>` の次にいきなり `<h5>` を使わない。`<h1>` はページに1回が基本。
- 画像が表示されない時は **パスの間違い** を疑う（`images/` や `../` の位置関係を確認）。
- `mailto:` の **コロン** を忘れない。
- `target="_blank"`（別タブ）は便利だが、開き方の選択をユーザーから奪う面があり使いすぎ注意。
- ファイル名・タグは **半角英数字・小文字** で統一（全角や日本語名は表示不具合の原因）。
