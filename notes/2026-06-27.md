# 今日学んだこと
### 2026-06-27｜第4章 シングルカラムのWebサイト制作スタート（4-1〜4-5途中）

**何をしたか**
- 第4章に入り、コンテンツを縦1列に並べる **シングルカラム（1カラム）レイアウト** の特徴（余白を活かした印象的なデザイン、モバイルとの相性の良さ）を学びました。
- これから制作する「WCB CAFE」サイトのページ構成（ロゴ・ナビゲーション・キャッチコピー・文章・ボタン・表）と、`head` → ロゴ/ナビ → コンテンツ → ファビコン という **制作の流れ** を把握しました。
- `index.html` の `<head>` 部分（`<meta charset>`・`<title>`・`<meta name="description">`）と、リセットCSS（`ress.css`）・Google Fonts（`Philosopher`）・自作CSS（`style.css`）の **読み込み** を記述しました。
- **モバイルファースト** の考え方と、`<meta name="viewport">` による表示領域の指定を学びました。
- Chromeの **デベロッパーツール** を使って、スマホなど別デバイスサイズで表示を検証する方法（右クリック→検証、デバイスツールバー、固定位置の変更）を学びました。
- `<header>` 部分のHTML（ロゴを `<h1>`、ナビを `<nav><ul><li>`）を書き、CSSの **共通部分**（`@charset`・`html`・`body`・`a`・`img`）と、ロゴの中央揃え・サイズ調整まで進めました。

**学んだ用語**

| 用語 | やさしい説明 |
|---|---|
| カラム | レイアウトを組む時の、縦に並んだ「列」のこと |
| カラムレイアウト | カラム（列）をページに垂直方向に区切って組んだレイアウト |
| シングルカラム（1カラム） | コンテンツを縦1列に並べるレイアウト。上から下へ一方向に視線を誘導でき、内容に集中させやすい。モバイルと相性が良い |
| マルチカラム | 複数のカラムを組み合わせたレイアウト。一覧表示など情報量が多いサイト向き（第5章で扱う） |
| ファビコン | ブラウザーのタブに表示される小さなアイコン |
| レスポンシブWebデザイン | デバイスの表示領域（画面幅）に応じて、Webページの表示を切り替える手法 |
| モバイルファースト | スマホ幅の装飾を **先に** 作り、デスクトップ用を後から足す制作方法。スマホ表示が速くなる利点 |
| デスクトップファースト | モバイルファーストの逆で、デスクトップ幅から先に作る方法。PC利用者が多いサイト向き |
| デベロッパーツール（検証ツール） | ブラウザーに標準搭載された、構成やCSSを確認・検証できる開発者向け機能。右クリック→「検証」で起動 |
| デバイスツールバー | デベロッパーツール左上の「2つの四角が重なったアイコン」。各デバイスサイズでの表示確認に使う |
| viewport（ビューポート） | 各デバイスにおける表示領域のこと |
| `<meta name="viewport">` | 表示領域の横幅を指定する `<head>` 内のタグ。これがないとスマホでデスクトップ幅のまま縮小表示される |
| `width=device-width` | `viewport` の値。表示幅を端末の画面幅に合わせる指定 |
| `initial-scale=1` | `viewport` の値。最初の表示倍率を等倍にする指定 |
| Google Fonts | Googleが無料提供するWebフォントのサービス。今回は見出し用に `Philosopher` を読み込む |
| `preconnect` | フォントなど外部サーバーへ事前接続して読み込みを速くする `<link>` の指定 |
| `<meta name="description">` | Webページの説明文を書くタグ。検索結果などで使われる |
| `@charset "UTF-8";` | CSSファイルの1行目に書く、文字化けを防ぐための文字コード宣言 |
| `font-size: 100%` | `html` に指定。16px固定ではなく、ブラウザー初期値やユーザー設定の文字サイズを正しく反映させる |
| `font-family` | 書体（フォント）を指定するプロパティ。複数指定すると、前から使える順に適用される |
| `line-height` | 行の高さ（行間）を指定するプロパティ |
| `text-decoration: none` | `<a>` タグの下線（標準の傍線）を消す指定 |
| `max-width: 100%`（img） | 画像が親要素より大きくならないようにする指定。はみ出し防止・レスポンシブの基本 |
| `text-align: center` | テキストやインライン要素を中央揃えにする指定 |
| `rem` | ルート要素（`html`）の文字サイズを基準とする単位。例：`.5rem` |
| `<header>` | ページ上部のロゴ・ナビなどを囲むタグ。多くは全ページ共通部分 |
| `<nav>` | ナビゲーションメニューを表すタグ |

**学んだ主な構文**

```html
<!-- ===== 4-3 head：文字コード・タイトル・説明文・CSS読み込み（c4-03-3） ===== -->

<!DOCTYPE html>
<html lang="ja">
  <head>
    <meta charset="utf-8">                  <!-- 文字コード -->
    <title>WCB Cafe</title>                 <!-- タブに出るタイトル -->
    <meta name="description" content="ブレンドコーヒーとヘルシーなオーガニックフードを提供するカフェ">

    <!-- リセット CSS（ブラウザー差をなくす。必ず最初に読み込む） -->
    <link rel="stylesheet" href="https://unpkg.com/ress/dist/ress.min.css">

    <!-- Google フォント（見出し用 Philosopher） -->
    <link rel="preconnect" href="https://fonts.googleapis.com">
    <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
    <link href="https://fonts.googleapis.com/css2?family=Philosopher&display=swap" rel="stylesheet">

    <!-- オリジナル CSS（自作。リセット・フォントの後＝いちばん下に書く） -->
    <link rel="stylesheet" href="css/style.css">
  </head>

  <body>

  </body>
</html>
```

```html
<!-- ===== 4-4 viewport：表示領域の横幅を端末に合わせる（c4-04-01） ===== -->

<meta name="viewport" content="width=device-width, initial-scale=1">
<!-- head 内に追記。スマホで文字が小さく潰れるのを防ぎ、端末幅に合わせて表示する -->
```

```html
<!-- ===== 4-5 header：ロゴ（h1）とナビゲーション（nav）（c4-05-1） ===== -->

<body>
  <header class="page-header">
    <h1 class="align-center">
      <a href="index.html"><img class="logo" src="images/logo.svg" alt="WCBカフェ ホーム"></a>
    </h1>
    <nav>
      <ul class="main-nav font-english">
        <li><a href="news.html">News</a></li>
        <li><a href="menu.html">Menu</a></li>
        <li><a href="contact.html">Contact</a></li>
      </ul>
    </nav>
  </header>
</body>
<!-- ロゴ画像は WCBCafe/images/ に保存。header はロゴ＋ナビをまとめる全ページ共通部分 -->
```

```css
/* ===== 4-5 style.css 共通部分（c4-05-2） ===== */

@charset "UTF-8";                            /* 1行目。文字化け防止 */

/* 共通部分
-------------------------------- */
html {
  font-size: 100%;                           /* 16px固定にせず、初期値/ユーザー設定を尊重 */
}
body {
  font-family: "Hiragino Kaku Gothic ProN", "Hiragino Sans", "BIZ UDPGothic", sans-serif; /* 書体 */
  line-height: 1.7;                          /* 行間 */
  color: #432;                               /* 文字色（3桁＝#443322の短縮） */
}
a {
  text-decoration: none;                     /* リンクの下線を消す */
}
img {
  max-width: 100%;                           /* 画像が親要素より大きくならないように */
}
```

```css
/* ===== 4-5 ロゴの中央揃えとサイズ・余白（c4-05-3） ===== */

/* レイアウト（共通クラスとして使い回す） */
.align-center {
  text-align: center;                        /* 中央揃え */
}

/* ヘッダー
-------------------------------- */
.page-header {
  padding-top: .5rem;                        /* ヘッダー上の余白（rem＝htmlの文字サイズ基準） */
}
.logo {
  width: 210px;                              /* ロゴの横幅。大きすぎる画像を縮小 */
}
```

**できるようになったこと**: シングルカラムレイアウトの特徴と用途の理解。WCB CAFEサイトの全体構成と制作手順の把握。`index.html` の `<head>` 記述（文字コード・タイトル・説明文）。リセットCSS・Googleフォント・自作CSSの正しい順序での読み込み。`<meta name="viewport">` によるモバイル表示対応。Chromeデベロッパーツールでの各デバイス表示の検証。`<header>`（ロゴ＋ナビゲーション）のHTML記述。共通CSS（文字サイズ・書体・行間・文字色・リンク装飾・画像の最大幅）の指定。共通クラスを使ったロゴの中央揃えとサイズ・余白の調整。

**つまずき・気づき**
- CSSの読み込み順は第3章3-18と同じく重要。**ress.css（リセット）→ Google Fonts → 自作 style.css** の順で、自作CSSを必ずいちばん下に書く（先に書くとリセットや上書きで効かなくなる）。
- `<meta name="viewport">` を書かないと、スマホでデスクトップ幅のまま縮小表示され、文字が小さく読みづらくなる。`width=device-width, initial-scale=1` で端末幅に合わせる。
- **モバイルファースト**＝モバイル幅の装飾を先に書き、後からデスクトップ用を足す進め方。逆はデスクトップファースト。閲覧ユーザーの利用環境で使い分ける。
- `html { font-size: 100%; }` は「16px固定」ではなく、ブラウザー初期値やユーザーが設定した文字サイズを尊重するための指定。
- `img { max-width: 100%; }` は画像が親要素からはみ出さないための基本指定。レスポンシブで特に重要。
- ロゴやナビなどの `<header>` は全ページ共通部分。`.align-center` のような **共通クラス** にしておくと他ページでも使い回せる。
- `color: #432` のような3桁の16進カラーは、6桁（`#443322`）の各桁を重ねた短縮表記。
- `rem` はルート（`html`）の文字サイズを基準にした単位（例 `.5rem`）。`px`（絶対値）との違いを意識する。
- デベロッパーツールが英語表示のときは、歯車（設定）→ Language → 「Japanese - 日本語」→ Reload DevTools で日本語化できる。
- 画像（`logo.svg` など）はサンプルデータの `images` フォルダーを `WCBCafe` 内に置いて利用する。HTMLの `src` のパスとフォルダー構成を一致させる必要がある。
