## 学習内容サマリー

### 第1章 最初に知っておこう！Webサイトの基本

#### 1-1 よいWebデザインとは
- デザインは「見栄え」ではなく **伝えるための手段**。何を伝えたいかが大切。
- 使いやすさを重視し、ユーザー目線で作る。
- デザインは **問題解決の手段**（例：海で破れない**プラスチック製紙幣**を1988年に世界初発行）。
- COLUMN：デザインの「センス」は才能ではなく **知識**。学べば誰でも身につく。

#### 1-2 様々な種類のWebサイト
| 種類 | 別名 | 特徴 |
|---|---|---|
| コーポレートサイト | ─ | 企業の公式サイト（会社概要・製品・採用） |
| プロモーションサイト | 特設サイト | 商品・サービス・イベントの告知。期間限定が多い |
| ポートフォリオサイト | ─ | 作品・実績を掲載（就職・受注に活用） |
| ショッピングサイト | ECサイト／オンラインストアサイト | ネット上で商品販売 |
| メディアサイト | ─ | ニュースや読み物記事（ブログもこの一種） |
| SNS | ソーシャル・ネットワーキング・サービス | リアルタイム交流・情報拡散 |

#### 1-3 ユーザビリティとは
- ユーザビリティ＝Webサイトの **使いやすさ**。
- 見やすいデザイン：配色の **コントラスト**確保／目立たせる要素の差別化／レイアウト統一。
- 読みやすい文章：**結論を先に**／専門用語を避ける／簡潔にまとめる。
- 使いやすい操作性：動作を予想できる／**動作を速く**（ファイルサイズ軽量化）／一目でリンクとわかる。
- COLUMN：コントラストは **グレースケール（白黒化）** で確認できる。

#### 1-4 Webサイトの仕組み
- **インターネット**＝世界中のコンピューターをつなぐネットワーク（仕組み）。
- **Web（World Wide Web）**＝インターネットの機能の1つ。
- **Webサーバー**（情報を公開・保管）と **Webクライアント**（受け取る側）が、**要求（リクエスト）→応答（レスポンス）** でやり取りしてページを表示する。
- **URLの構成**：`http://example.com/sample/index.html`

| パーツ | 例 |
|---|---|
| プロトコル | `http://` |
| ドメイン名 | `example.com` |
| ディレクトリ名 | `sample/` |
| ファイル名 | `index` |
| ファイル拡張子 | `.html` |

#### 1-5 デバイスの種類
- **① 端末**：iOS／Androidデバイス、モバイル／スマート／ウェアラブル／IoTデバイス。
- **② 周辺機器**：USB／ストレージ／オーディオデバイス。
- COLUMN：「不明なデバイス」＝接続不良・利用不可。**デバイスドライバー**やケーブル・電源を確認。

#### 1-6 ブラウザーの種類
- Webページのデータは記号の並んだ「コード」。**ブラウザー**が解読し、見やすく表示する。
- 主なブラウザー：**Chrome／Safari／Microsoft Edge／Firefox**。
- 表示確認用の **標準ブラウザー** を決めておく。
- 日本のシェア（2023年1〜6月／StatCounter）：Chrome 49.9% ＞ Safari 29.84% ＞ Edge 13.33%。

#### 1-7 制作の流れ（6工程）
| No. | 工程 | 内容 |
|---|---|---|
| 01 | 企画（PLAN） | 目的・メイン/サブゴール・ターゲットユーザー・**ペルソナ**を整理 |
| 02 | サイトマップ（SITE MAP） | 必要ページとリンク関係を図に。**2〜3階層以内**に抑える |
| 03 | ワイヤーフレーム（WIREFRAME） | 骨組みを線とボックスで。**ファーストビュー**と優先順位、視線（上→下・左→右）を意識 |
| 04 | デザイン（DESIGN） | **デザインカンプ**（完成見本）を作り込む |
| 05 | コーディング（CODING） | **HTML/CSS/画像**ファイルを用意して実装 |
| 06 | 公開（WEBSITE） | サーバーへアップロードし表示・リンクを最終確認 |

- 作成ツール例：Figma／Cacoo／Moqups／Adobe Illustrator／Sketch（Mac専用） など。※Adobe XD は現在メンテナンスモードで新規インストール不可。
- COLUMN：AIによる自動Webデザインは高速・低コストだがテンプレート依存。AIは制作の **「副操縦士」**、最終判断は人間。

### 第2章 Webの基本構造を作る！HTMLの基本

#### 2-1 HTMLとは
- **概念**：HTML＝**Hyper Text Markup Language**の略。Webページの**土台となる言語**。
- **ルール**：表示したい文章などを `<` と `>` で挟んだ **タグ** で囲み、「ここは見出し」「これはリンク」などの**意味**をコンピューターに指示する。
- **具体例**：文章を `<h1>` で囲めば「見出し」、`<a>` で囲めば「リンク」として解釈される。

#### 2-2 HTMLファイルを作ろう
- **概念**：テキストエディター（VS Code）で書き、`index.html` として保存 → ブラウザーで開くと表示される。
- **ルール**：HTMLのファイル名には守るべき決まりがある。

| ルール | 良い例 | 悪い例 |
|---|---|---|
| 拡張子 `.html` をつける | `mypage.html` | `mypage`（拡張子なし） |
| 半角英数字のみ（日本語・全角不可） | `my-page.html` | `マイページ.html`／`ｍｙｐａｇｅ.html` |
| 使えない記号を避ける（`\ : , ; " < > \| *` 等。`/` はフォルダ区切り用） | `my-page.html`／`my_page.html` | `my*page.html`／`my/page.html` |
| 空白（スペース）を入れない | `my-page.html` | `my page.html` |
| 小文字に統一する | `mypage.html` | `MyPage.html`／`mypage.HTML` |
| トップページは `index.html` にする | `index.html` | （URL省略が効かなくなる） |

- 補足：使ってよい記号は基本的に `-`（ハイフン）と `_`（アンダースコア）のみ。

#### 2-3 HTMLファイルの骨組み
| タグ／記述 | 役割 |
|---|---|
| `<!doctype html>` | **Doctype宣言**。HTMLのバージョン・仕様を示す（本書は現主流の **HTML Living Standard** 準拠） |
| `<html lang="ja">〜</html>` | HTML文書であることを示す。`lang="ja"` で**日本語の文書**だと指定 |
| `<head>〜</head>` | ページの情報を記述する領域。**ブラウザーには表示されない** |
| `<meta charset="UTF-8">` | **文字コード**を UTF-8 に指定（文字化け防止） |
| `<title>〜</title>` | タブ・ブックマーク・検索結果に出る**ページタイトル** |
| `<meta name="description" content="〜">` | ページの**説明文**（検索結果にタイトルと共に表示される） |
| `<body>〜</body>` | HTML文書の**本体**。ここに書いた内容が**実際に表示される** |

- ポイント：`<head>` には「ページの情報」、`<body>` には「表示したいコンテンツ」を書く。
- COLUMN（文字コード）：以前主流だった `Shift_JIS` は iOS 等で文字化けする。現在は世界中で使われる **UTF-8** がほぼ標準。

#### 2-4 HTMLの基本の書き方
- **タグの構成**：`<タグ名>コンテンツ内容</タグ名>` で1つの **要素**。前半が**開始タグ**、`/` 付きが**終了タグ**。終了タグのないタグもある（例：`<img>`）。
- **書く時のルール**：半角英数字で書く（全角不可）／大文字・小文字の区別は基本ないが**小文字で統一**するとよい。
- **入れ子（ネスト）**：タグの中にタグを入れられる。**手前に開いたタグから順に閉じる**。
  - 良い例：`<p><strong>入れ子</strong>の書き方</p>`
  - 悪い例：`<p><strong>入れ子</p>の書き方</strong>`（交差してNG）
- **属性と値**：開始タグ内に付加情報を書く。`属性名="値"` の形で、値は**ダブルクォーテーション**で囲む（例：`<a href="about.html">`）。

#### 2-5 見出しをつけよう
- `<h1>`〜`<h6>` の6種類（`h`＝**heading**）。`<h1>` が最大、数字が大きいほど小さい見出し。
- **使う順番のルール**：`<h1>` から順に使う（見た目目的で突然 `<h5>` を使わない）／`<h1>` は**1ページ原則1回**／多くは `<h4>` 程度まで。

#### 2-6 文章を表示しよう
- `<p>`＝**Paragraph**（段落）。最もよく使うタグの1つ。
- `<p>` が連続すると**自動で改行**され段落間にスペースが入る。長文は段落で分けると読みやすい。

#### 2-7 画像を挿入しよう
- `<img>`＝画像タグ。**終了タグがなく単独で使う**。
- 必須属性：

| 属性 | 役割 |
|---|---|
| `src` | 表示する画像の場所（ファイルパス）を指定 |
| `alt` | 画像が読み込めない時の**代替テキスト**。画像の意味を伝える |

- **ファイルパス（相対パス）**：

| 位置関係 | 書き方 | 例 |
|---|---|---|
| 同じフォルダ | ファイル名のみ | `sakura.jpg` |
| 1つ下のフォルダ内 | `フォルダ名/ファイル名` | `images/sakura.jpg` |
| 1つ上の階層へ | `../` で上がる | `../images/sakura.jpg` |

- **絶対パス**：別サイトの画像など。`http://`／`https://` から始まる完全なURLで書く。
- COLUMN（読みやすさ）：1段落 **3〜5行**、1行 **30〜45文字** 程度が目安。

#### 2-8 リンクをはろう
- `<a href="">〜</a>`＝リンク。リンク先は **href属性** で指定。別サイトへは `http://`／`https://` を**先頭に必ず**付ける。
- 例：`今すぐ<a href="https://google.co.jp">Google</a>で検索！`（デフォルトは青文字＋下線）。
- **画像にリンク**：`<img>` を `<a>` で囲む。
- **メール宛て**：`<a href="mailto:info@example.com">お問い合わせ</a>`。
- COLUMN（別タブ）：`target="_blank"` で別タブに開く。ただしUX上は賛否あり、使いすぎ注意。

#### 2-9 リストを表示しよう
| リスト種類 | 親タグ | 略 | 表示 |
|---|---|---|---|
| 箇条書き（順序なし） | `<ul>` ＋ `<li>` | Unordered List | 先頭に**黒丸（・）** |
| 番号付き（順序あり） | `<ol>` ＋ `<li>` | Ordered List | 先頭に**連番（1. 2. 3.）** |

- 共通：項目は `<li>`（List Item）で囲み、親タグの中に必要な数だけ並べる。
- COLUMN：右クリック →「ページのソースを表示」で、どのWebサイトのHTMLも閲覧でき制作の参考になる。

#### 2-10 表を作ろう
- **概念**：時間割や料金表など、様々な表を作成するために `<table>` タグを使う。複数のタグを組み合わせて使う。

| タグ | 目的 |
|---|---|
| `<table>` | 表全体を囲む |
| `<tr>` | 「Table Row」の略。表の**1行**を囲む |
| `<th>` | 「Table Header」の略。表の**見出しとなるセル**を作成 |
| `<td>` | 「Table Data」の略。表の**データとなるセル**を作成 |

- **ルール**：`<table>` 内に `<tr>` で行を追加し、その中に `<th>` または `<td>` でセルを作る。`<tr>` 内のセルは同じ数にしないとレイアウトが崩れる。見出しがない場合、`<th>` タグは省略できる。
- **補足**：デフォルトは仕切り線がないため、わかりやすくするために `<table>` タグに `border="1"` を加えることがある（本格的な装飾は第3章のCSSで行う）。

##### セルをつなげる
- **概念**：複数のセルをつなげて1つのセルとして表示できる。

| 属性 | 方向 | 用途 |
|---|---|---|
| `colspan="数値"` | 横方向（列をまたぐ） | `<th>` または `<td>` に追加。つなげたいセルの数を指定。つながれたセルのタグを削除する |
| `rowspan="数値"` | 縦方向（行をまたぐ） | `<td>` に追加。つなげたいセルの数を指定。縦に並んだセルのタグを削除する |

- COLUMN（コメントアウト）：`<!-- コメント -->` でHTMLのコード内にコメントを残せる。ブラウザーには表示されない。複数行でも書ける。一時的に一部のコードを非表示にしたい時にも使える。

#### 2-11 フォームを作ろう
- **概念**：お問い合わせ・検索ボックス・会員登録など、ユーザーがテキストを入力したり選択したりするフォーム。HTMLのタグで必要なパーツを組み合わせて作成する。
- **ポイント**：`<form>` タグはフォームの表示部分を担当。実際にデータを送信・処理するにはPHPなどのプログラムとの連携が必要（本書はHTML/CSSの記述範囲を扱う）。

##### `<form>` タグ（フォーム欄を作る）
- フォームで使用するすべてのパーツを `<form>` タグで囲む。

| 属性 | 用途 |
|---|---|
| `action` | データの送信先ページを指定 |
| `method` | データの転送方法の指定。主に `get` か `post` を入力 |
| `name` | フォームの名前を指定 |

##### `<input>` タグ（フォームで使うパーツ）
- フォーム内で使う各パーツを設置するタグ。`type` 属性で用途に応じて表示するパーツを変える。

| `type` 属性値 | 表示されるパーツ | 補足 |
|---|---|---|
| `text` | 1行テキスト入力欄（初期値） | `placeholder` 属性であらかじめ入力欄にテキストを表示できる |
| `search` | 検索する時のテキスト入力欄 | ─ |
| `email` | メールアドレス入力欄 | ブラウザーが書式チェックしてくれる |
| `tel` | 電話番号入力欄 | ─ |
| `url` | WebサイトのURL入力欄 | ─ |
| `radio` | ラジオボタン（1つだけ選択） | `name` 属性を同じ値にするとグループになり1つのみ選択可。`checked` 属性で最初から選択状態にできる |
| `checkbox` | チェックボックス（複数選択可） | `name` 属性で同グループに。`checked` 属性で最初から選択状態にできる。趣味・利用目的など複数回答の時に利用 |
| `submit` | 送信ボタン | `value` 属性でボタン上のテキストを指定。文字サイズ・背景色はCSSで変更可 |
| `image` | 画像ボタン | `src` 属性で画像ファイルを指定。`alt` 属性で説明テキストを指定 |

##### `<select>` タグ＋`<option>` タグ（セレクトボックスを作る）
- クリックすると選択肢が表示されるパーツ。都道府県選択などに多く使われる。

| タグ・属性 | 用途 |
|---|---|
| `<select name="名前">` | セレクトボックス全体を囲む |
| `multiple` 属性（selectに付与） | Shift または Ctrl キーで複数選択可能にする |
| `<option value="値">` | 各選択肢を囲む。`value` は送信される値 |
| `selected` 属性（optionに付与） | 最初から選択された状態にする |

##### `<textarea>` タグ（複数行テキスト入力欄を作る）
- 複数行にわたるテキストを入力できる。お問い合わせ内容やメッセージ入力時によく使われる。
- タグで囲まれた部分が初期値として表示される。ただし入力欄をクリックしても消えないため、初期表示テキストは `placeholder` 属性で指定するとよい。
- COLUMN（フォームを動作させるにはプログラミングが必要）：フォームの各パーツをHTMLで表示することはできるが、実際に情報を送信するにはPHPなどのプログラムとの連携が必要。メールアドレス宛のリンクで対応する方法もある（p.282参照）。

#### 2-12 より使いやすいフォームにしよう
- **概念**：テキスト入力欄の横に「名前」「電話番号」などのテキストを表示させるものを**ラベル**と呼ぶ。`<label>` タグでフォームのパーツとラベルテキストを関連付けると、小さなラジオボタンやチェックボックスをクリックしなくてもテキスト部分のクリックで選択できるようになり、使いやすいフォームになる。

##### `<label>` タグの使い方
- **ルール**：ラベルテキストの部分を `<label>` タグで囲み、`for` 属性をつける。関連付けたいフォームのパーツには `id` 属性をつける。`for` 属性と `id` 属性の値（識別名）を同じにすることでフォームとパーツが関連付けられる。
- **注意**：識別名（`id` 属性の値）は必ず一対になる必要があり、同じファイル内で1ヶ所にのみ使用できる（重複不可）。

#### 2-13 グループ分けをしよう（第2章ラスト）
- **概念**：細かい要素を上から並べるだけでは整ったレイアウトにならない。意味のあるまとまりごとにタグで囲んで**グループ化**する。グループ分け用タグには「この部分はこういう内容だ」とコンピューターに伝える役割もある。
- **ポイント**：HTMLだけでは見た目は変わらないが、グループ分けしておくと第3章のCSSで色やレイアウトを設定しやすくなる。

| タグ | 役割 |
|---|---|
| `<header>` | ページ上部のかたまり（ロゴ・タイトル・ナビなど）。`<head>` 要素とは別物 |
| `<nav>` | メインのナビゲーションメニュー。`<header>` の中に入ることが多い |
| `<main>` | ページの核となるメインコンテンツ全体 |
| `<article>` | それ単体で独立して成り立つ記事（ニュース・ブログ記事など） |
| `<section>` | 1つのテーマを持つかたまり。`<article>` と違い単体では完結しない |
| `<aside>` | 本文ではない補足情報（サイドバーなど） |
| `<footer>` | ページ下部のかたまり（コピーライト・SNSリンクなど） |
| `<div>` | 意味を持たないタグ。他に適切なタグがない時の最終手段 |

### 第3章 Webのデザインを作る！CSSの基本

#### 3-1 CSSとは
- **概念**：HTMLのみのWebサイトは白背景に黒文字だけ。CSS＝**Cascading Style Sheets** の略で、土台であるHTMLの**見た目を装飾する言語**。拡張子は `.css`。
- **具体例**：装飾済みのサイトでCSSを無効化すると、背景・レイアウト・文字装飾がすべて消えてシンプルな文書に戻る。CSSがいかに見た目を大きく変えているかがわかる。

#### 3-2 CSSを適用させる方法（3通り）
| 方法 | 書き方 | 特徴 |
|---|---|---|
| ① 外部CSSファイル | `<head>` 内に `<link rel="stylesheet" href="style.css">` | **最も一般的**。1つのCSSを複数HTMLで共有でき一括管理しやすい |
| ② `<style>` タグ | `<head>` 内に `<style>` を書きCSSを記述 | そのHTMLファイル内だけに適用される |
| ③ `style` 属性 | HTMLタグ内に `style="..."` を直接記述 | そのタグだけに適用。**優先順位が最も高い**が管理しづらい |

- **ポイント**：特別な理由がなければ ① の外部CSSファイルを使う。

#### 3-3 CSSファイルを作ろう
- **ルール**：CSSファイルの**1行目に必ず `@charset "UTF-8";` を書く**（文字化け防止）。これより前にコードを書くとエラーになる。
- 作成した `style.css` を、HTMLの `<head>` 内に `<link>` タグを追加して読み込ませる。

#### 3-4 CSSの基本の書き方
- **基本文法**：`セレクター { プロパティ: 値; }` の3要素で「どの部分の・何を・どう変えるか」を指定する。

| 要素 | 役割 |
|---|---|
| セレクター | **どの部分**を装飾するか（タグ名・クラス・IDなど） |
| プロパティ | **何を**変えるか（文字色・文字サイズなど） |
| 値 | 見た目を**どう**変えるか（赤色にする・20pxにするなど） |

- **記述ルール**：半角英数字で書く／小文字で統一すると無難／複数セレクターは `,`（カンマ）で区切る／複数の装飾は値の最後に `;`（セミコロン）を付けて区切る。
- **単位**：`px`（絶対値・1ピクセル基準）／`%`（親要素基準の相対値）／`rem`（ルート要素＝`<html>`基準の相対値）。値が `0` 以外は単位を必ず付ける。
- **子孫セレクター**：セレクターを半角スペースで区切ると「要素の中の要素」だけに指定できる（例：`div p` は `<div>` 内の `<p>` のみ）。

#### 3-5 文字や文章を装飾しよう
| プロパティ | 役割 | 主な値 |
|---|---|---|
| `font-size` | 文字の大きさ | `px`・`rem`・`%`、`xx-small`〜`xx-large`（`medium`が標準） |
| `font-family` | フォントの種類 | カンマ区切りで先頭から適用。日本語名・スペース入りは `" "` で囲み英語名も併記 |
| `font-weight` | 文字の太さ | `normal`／`bold`／`lighter`／`bolder`／1〜1000 |
| `line-height` | 行の高さ（行間） | 単位なしの数値が好まれる。おすすめ 1.5〜1.9 |
| `text-align` | テキストを揃える位置 | `left`／`right`／`center`／`justify`（両端揃え） |

- **デザインの考え方**：本文は **14〜18px** 程度／フォントは **1〜3種類**まで／装飾系フォント・太字は見出しや要所のみ／中央揃えは2〜3行の短文向き（長文は左揃えか両端揃え）／見出しと本文の比率＝**ジャンプ率**が高いほど躍動的、低いほど上品。
- **フォントの種類**：明朝体（serif系・長文向き）／ゴシック体（sans-serif系・読みやすく見出し向き）／装飾系（本文には使わない）。

#### 3-6 Webフォントを使おう
- **概念**：**Webフォント**＝フォントファイルをWebサーバー上に置き、ユーザーがインストールしていなくても表示できる技術。
- **実装**：無料の **Google Fonts** が手軽。使いたいフォントを選び、発行されたコード（`preconnect` の `<link>` を含む）を `<head>` 内にまとめて貼り付け、CSS側で適用したい要素に `font-family` を指定する。

#### 3-7 色をつけよう
| 指定方法 | 書き方 | 例 |
|---|---|---|
| カラーコード | `#` ＋ 3桁または6桁の英数字（左2=赤・中2=緑・右2=青）。同じ数値の並びは3桁に省略可 | `#ffffff`（白）／`#f00`（赤） |
| RGB値 | `rgb(赤, 緑, 青)`（各0〜255） | `rgb(255,255,255)`（白） |
| Alpha値（rgba） | `rgba(赤, 緑, 青, 透明度)`（透明度0〜1、0=透明） | `rgba(255,255,255,.5)`（半透明白） |
| 色の名前 | 決められた色名 | `red`／`blue`／`tomato` など |

- 文字色は `color`、背景色は `background-color` で指定。ページ全体の背景は `<body>` に指定する。
- **無彩色**（白・グレイ・黒）はカラーコードの赤・緑・青の3組が同じ値（例：`#9a9a9a`）。

#### 3-8 上手に配色しよう（デザイン理論）
- **色の3要素**：色相（赤・黄・緑などの色みの違い／反対が補色・隣りが類似色相）・明度（明るさ）・彩度（鮮やかさ）。
- **色の温度**：暖色（赤中心・暖かい）／寒色（青中心・涼しい）／中性色（緑・紫など）／無彩色（白・灰・黒）。
- **配色の比率**：ベースカラー **70%**（背景）・メインカラー **25%**（テーマ色）・アクセントカラー **5%**（ボタンなどの目立たせたい部分）が基本の目安。

> 3-9（配色例のカテゴリー集）は今回の学習画像に含まれていないため未収録。

#### 3-10 背景を彩ろう
| プロパティ | 役割 | 主な値 |
|---|---|---|
| `background-image` | 背景に画像を設置 | `url(画像のパス)`／`none` |
| `background-repeat` | 繰り返し方 | `repeat`／`repeat-x`（横）／`repeat-y`（縦）／`no-repeat` |
| `background-size` | 画像の大きさ | `cover`（領域を埋める・はみ出しは切れる）／`contain`（全体表示・余白が出る）／数値 |
| `background-position` | 表示開始位置 | 「横方向 縦方向」の順。初期値 `left top` |
| `background` | 上記をまとめる一括指定 | `size` は `position` の直後に `/` で区切る |

- 画像が読み込めない時に備え、背景画像と同時に近い色の背景色も指定しておくと安心。

#### 3-11 幅と高さを指定しよう
- `width`（幅）・`height`（高さ）で要素の大きさを指定。値は数値か `auto`。**インライン要素（`<a>`・`<span>`）には効かない**。
- `width` を `%` で指定すると親要素の幅に対する比率になる。ブロック要素は初期値 `auto` で親の幅いっぱいに広がる。

| 単位 | 種類 | 基準 |
|---|---|---|
| `px` | 絶対単位 | 表示領域や親に影響されない |
| `%` | 相対単位 | 親要素のサイズ |
| `em` | 相対単位 | 親要素のフォントサイズ（16pxなら 1em=16px） |
| `rem` | 相対単位 | ルート要素（`<html>`）のフォントサイズ |
| `vw`／`vh` | 相対単位 | ビューポート（表示領域）の幅／高さ。`50vw`＝表示幅の半分 |

#### 3-12 余白を調整しよう
- **ボックスの概念**：要素は「margin（外側の余白）＞ border（枠線）＞ padding（内側の余白）＞ 中身（width/height）」という入れ子構造。

| プロパティ | 役割 |
|---|---|
| `margin` | 要素の外側の余白（他の要素との距離） |
| `padding` | 要素の内側の余白（枠線から中身までの距離） |

- **四辺の一括指定**（時計回り）：値1個=四辺すべて／2個=上下・左右／3個=上・左右・下／4個=上・右・下・左。`-top`／`-bottom`／`-left`／`-right` で個別指定も可能。
- **デザインの考え方**：枠と文字の間（padding）は**文字サイズの1〜1.5倍**が目安／関連する情報は近づけ、関連しない情報は離す「近接」でグループを伝える。
- COLUMN：CSSのコメントは `/* */`（HTMLの `<!-- -->` とは異なる）。複数行可、一時的な無効化にも使える。

#### 3-13 線を引こう
| プロパティ | 役割 | 主な値 |
|---|---|---|
| `border-width` | 線の太さ | `thin`／`medium`／`thick`／数値 |
| `border-style` | 線の種類 | `none`／`solid`／`double`／`dashed`／`dotted`／`groove`／`ridge`／`inset`／`outset` |
| `border-color` | 線の色 | カラーコード・RGB値・色名（未指定だと黒） |
| `border` | 上記をまとめる一括指定 | 順不同。`border-top`／`-bottom`／`-left`／`-right` で辺ごとに指定可 |

- **注意**：`border-style` の初期値は `none`。太さや色だけ指定しても線は出ないので、`border-style` を必ず一緒に指定する。

#### 3-14 リストを装飾しよう
| プロパティ | 役割 | 主な値 |
|---|---|---|
| `list-style-type` | マーカーの種類 | `none`／`disc`（黒丸）／`circle`（白丸）／`square`（黒四角）／`decimal`（数字）／`hiragana`・`cjk-ideographic`（漢数字）など |
| `list-style-position` | マーカーの位置 | `outside`（外側）／`inside`（内側） |
| `list-style-image` | マーカーに画像を使う | `url(画像のパス)`／`none` |
| `list-style` | 上記をまとめる一括指定 | type と image を両方指定すると image が優先 |

- 未指定だと `<ul>` は黒丸、`<ol>` は数字。`list-style-type` は `<ul>`／`<ol>` に指定する。

#### 3-15 クラスとIDを使った指定方法
- **概念**：同じタグの一部だけ装飾を変えたい時、クラスやIDを付けてピンポイントで指定する。

| | CSSの書き方 | HTML属性 | 使用回数 |
|---|---|---|---|
| クラス（class） | `.クラス名` | `class="名前"`（`.` は付けない） | 1ページ内で**何度でも** |
| ID（id） | `#ID名` | `id="名前"`（`#` は付けない） | 1ページ内で**1度だけ** |

- `タグ名.クラス名`（例：`p.blue`）と書くと、そのタグかつそのクラスの箇所だけに適用される。
- 1つのタグに複数クラス（半角スペース区切り）やID＋クラスの併用も可能。
- **優先順位**：同じタグにクラスとIDで違う装飾を指定すると**IDが優先**される。
- **命名ルール**：①空白を入れない ②英数字・ハイフン（`-`）・アンダースコア（`_`）で書く ③1文字目は必ず英字。
- COLUMN：ID属性は `<a href="#contents">` のようにページ内リンクのジャンプ先にも使える。

#### 3-16 レイアウトを組もう（Flexbox）
- **概念**：**Flexbox**（Flexible Box Layout Module）は複雑なレイアウトも簡単に組める仕組みで、現在の主流。親要素（Flexコンテナー）に `display: flex;` を指定すると、中の子要素（Flexアイテム）が横並びになる。

| プロパティ（親に指定） | 役割 | 主な値 |
|---|---|---|
| `display: flex` | 子要素を横並びにする起点 | ─ |
| `flex-direction` | 子要素の並ぶ向き | `row`（初期値・左→右）／`row-reverse`／`column`（上→下）／`column-reverse` |
| `flex-wrap` | 折り返しの有無 | `nowrap`（初期値・1行）／`wrap`（上→下に折り返し）／`wrap-reverse` |
| `justify-content` | 水平方向の揃え | `flex-start`（初期値・左）／`flex-end`（右）／`center`／`space-between`／`space-around` |

##### 垂直方向の揃え（`align-items`）／複数行の揃え（`align-content`）
- **概念**：`justify-content` が水平方向の揃えなのに対し、`align-items` は子要素を**縦方向（垂直方向）**のどこに置くかを指定する。親要素に縦の空きスペースがある時に効く。

| `align-items` の値 | 内容 |
|---|---|
| `stretch` | 初期値。親（または一番背の高い子）の高さに合わせて子要素を引き伸ばして配置 |
| `flex-start` | 親要素の開始位置（上）から配置。上揃え |
| `flex-end` | 親要素の終点（下）から配置。下揃え |
| `center` | 中央揃え |
| `baseline` | 文字のベースラインで揃える |

- **概念**：`align-content` は子要素が**複数行に折り返した時**の、行全体の縦方向の揃えを指定する。`flex-wrap: nowrap`（1行）が適用されていると**無効**になる。

| `align-content` の値 | 内容 |
|---|---|
| `stretch` | 初期値。親の高さに合わせて広げて配置 |
| `flex-start` | 親の開始位置から配置。上揃え |
| `flex-end` | 親の終点から配置。下揃え |
| `center` | 中央揃え |
| `space-between` | 最初と最後の行を上下端に置き、残りを均等な間隔で配置 |
| `space-around` | 上下端にある行も含め、均等な間隔で配置 |

- **補足**：`align-items: center` と `height: 100vh`（表示領域の高さ基準）を併用すると、要素を画面の縦中央に置ける。

#### 3-17 CSSグリッドでタイル型に並べよう
- **概念**：**CSSグリッド**は、同じ大きさのボックスを格子状（タイル型）に並べるレイアウト手法。Flexbox と同様に親要素（**グリッドコンテナー**）で全体を囲み、その中の子要素（**グリッドアイテム**）を配置する。アイテム間の空白は**グリッドギャップ**と呼ぶ。
- **ルール**：親要素に `display: grid;` を指定して開始する。これだけだと子要素は縦に並ぶだけで、横並びには列幅の指定が必要。

| プロパティ（親に指定） | 役割 | 主な値 |
|---|---|---|
| `display: grid` | 要素をグリッドコンテナーにする起点 | ─ |
| `grid-template-columns` | 各列（横方向）のアイテムの**幅**を指定 | 半角スペース区切りで列数だけ並べる（例：`200px 200px 200px`／`1fr 1fr 1fr`） |
| `grid-template-rows` | 各行（縦方向）のアイテムの**高さ**を指定 | 半角スペース区切りで行数だけ並べる（例：`200px 200px`） |
| `gap` | グリッドアイテム同士の**余白** | 数値（例：`10px`）。上下左右端はコンテナーの枠に揃う。**Flexbox でも使える** |

- **`fr` 単位**：「fraction（比率）」の略で、幅を具体的な数値ではなく**割合**で指定する。`px`（固定）は画面が広いと右に余白ができ、狭いと見切れるが、`fr` を使うと画面幅に合わせて自動で伸縮する（例：`1fr 1fr 1fr` で 1:1:1）。

##### 読みやすいレイアウトとは（レイアウト設計の考え方）
- **視線の流れ**：ユーザーの視線には法則がある。
  - **Zの法則**：左上→右上→左下→右下と Z の字に動く。初めて訪れる／画像の多いサイトで全体を見渡す時に多い。
  - **Fの法則**：左上から右へ、下へ下へと F の字に動く。再訪／情報量の多いサイトで目的の情報を探す時に多い。
- **情報の優先順位**：Webページは画面の**左上から読まれ始める**ため、重要な情報・注目させたい要素は左上に置く。優先順位の低いものは下部や右側へ。
- **配置の工夫**：目立たせたい情報は**面積を大きく**すると注目度が上がる（例：トップのメインビジュアル）。重要でない情報は小さくし、メリハリをつける。
- **ポイント**：レイアウトは **Flexbox や CSSグリッド** を使って組む。決める時は優先順位を考えて配置する。

#### 3-18 デフォルトCSSをリセットしよう
- **概念**：ブラウザーは標準で独自の**デフォルトCSS**（余白・フォント・文字サイズなど）を当てており、ブラウザーごとに表示差が出る。**リセットCSS**を使うと、ブラウザー本来のCSSを打ち消し、異なるブラウザーでも表示を統一できる。
- **本書で使うリセットCSS**：`ress.css`（外部公開ファイル）。すべてのデフォルトを消すのではなく、**デフォルトを活かしつつブラウザー間の最小限の誤差をなくす**タイプ。
- **読み込ませ方**：`<head>` 内に `<link>` で読み込む。ファイルをダウンロードしてもよいが、Web上のURL（`https://unpkg.com/ress/dist/ress.min.css`）を直接書いても適用できる（**インターネット接続が必要**）。

| 読み込み順 | 結果 |
|---|---|
| ✅ 良い例：`ress.css` を**先**、自作CSSを**後** | デフォルトCSSを無効化した上で自作CSSを反映できる |
| ❌ 悪い例：自作CSSを**先**、`ress.css` を**後** | 後から読まれた `ress.css` が優先され、自作CSSまでリセットされる |

- **補足**：`ress.css` を読み込むとフォームの枠線や余白が消え、ChromeとFirefoxなど異なるブラウザーで表示が統一される。これをベースに自作CSSで目的のデザインへ変更していく。
- COLUMN：Flexbox には他にも様々な指定方法があり、Webクリエイターボックスの「CSS Flexbox チートシート」が参考になる。

### 第4章 シングルカラムのWebサイトを制作する

> この章から「WCB CAFE」という1つのWebサイトを、ステップごとに完成させていく。まずはシングルカラムのホームページから。

#### 4-1 シングルカラムとは
- **概念**：縦に並んだ列を**カラム**、それを垂直方向に区切って組んだものを**カラムレイアウト**という。1カラムで作るのが**シングルカラム**。
- **特徴**：余白を十分に確保でき、大きな画像で印象的なデザインを作れる。上から下への一方向に視線を誘導でき、内容に集中させやすい。**画面の小さなモバイルデバイスと相性が良い**。
- **POINT**：一度に表示できる情報量が限られるため、アイテム数の多いショッピング・ギャラリーサイトには不向き。その場合は複数カラムの**マルチカラム**（第5章）を使う。

#### 4-2 シングルカラムページの制作の流れ
- **作成するページ**：Webサイトの顔となるホームページ。大きな画像を一面に配置し、ロゴ `<h1>`・ナビ `<nav><ul><li>`・キャッチコピー `<h2>`・文章 `<p>`・ボタン `<a>`・背景画像 `<div>` で構成する。
- **制作の流れ**：

| 手順 | 内容 |
|---|---|
| 01 | 「head」部分の記述（ページ情報） |
| 02 | ロゴ・ナビゲーションメニューの作成（**全ページ共通**部分） |
| 03 | コンテンツ部分の作成（キャッチコピー・文章・ボタン・表。ボタンにはホバー時のアニメーションを付ける） |
| 04 | ファビコンの設定（タブに表示される小さなアイコン） |

#### 4-3 「head」を記述しよう
- **ファイル準備**：フォルダー `WCBCafe` を作り、中に `index.html` を新規作成。さらに `css/style.css`、`images/`（ロゴ等）を用意する。
- **head の中身**：`<meta charset="utf-8">`（文字コード）／`<title>`（タブのタイトル）／`<meta name="description">`（ページ説明文）。
- **CSSの読み込み（順番が重要）**：

| 順 | 読み込むもの | 役割 |
|---|---|---|
| 1 | `ress.css`（リセットCSS） | ブラウザー差をなくす。必ず最初 |
| 2 | Google Fonts（`Philosopher`） | 見出し用フォント。`preconnect` で事前接続 |
| 3 | `css/style.css`（自作） | 最後に書く。先頭に置くとリセット等で上書きされる |

#### 4-4 モバイルファーストで作成する準備
- **モバイルファースト**：表示領域に応じて表示を切り替える手法を**レスポンシブWebデザイン**という。本書はまずモバイル幅で作り、後からデスクトップを足す**モバイルファースト**で進める（スマホ表示が速くなる利点）。逆は**デスクトップファースト**。
- **viewport**：表示領域を端末幅に合わせる指定。これがないとスマホでデスクトップ幅のまま縮小表示され読みづらい。
  - `<meta name="viewport" content="width=device-width, initial-scale=1">`
- **確認方法**：Chromeの**デベロッパーツール**（右クリック→「検証」、または `Cmd/Ctrl + Shift + C`）。左上の「2つの四角が重なったアイコン」でデバイスサイズを切り替え。パネルの3点メニューから固定位置（左／下／右／別ウィンドウ）を変更できる。英語表示は歯車→Language→日本語で切替。

#### 4-5 「header」部分を作ろう
- **HTML**：`<body>` 内に `<header class="page-header">` を置き、`<h1 class="align-center">` にロゴ画像（`<a>` で囲んだ `<img class="logo">`）、`<nav>` 内の `<ul class="main-nav font-english">` に News／Menu／Contact のリンクを記述。ヘッダーは全ページ共通部分。
- **CSS共通部分**：`@charset "UTF-8";`（1行目・文字化け防止）／`html { font-size: 100%; }`（初期値・ユーザー設定を尊重）／`body` に書体・`line-height`・文字色／`a { text-decoration: none; }`（下線消去）／`img { max-width: 100%; }`（はみ出し防止）。

| プロパティ | 役割 |
|---|---|
| `text-align: center`（`.align-center`） | ロゴなどを中央揃え（共通クラスとして使い回す） |
| `padding-top: .5rem`（`.page-header`） | ヘッダー上の余白（`rem`＝`html`の文字サイズ基準） |
| `width: 210px`（`.logo`） | 大きすぎるロゴ画像の横幅を調整 |

#### 4-5（続き）ロゴのサイズとナビゲーションの装飾
- **ロゴ**：`.logo { width: 210px; }` でロゴ画像を縮小し、`.align-center`（`text-align: center`）で中央寄せ。共通クラスとして他ページでも使い回す。
- **欧文フォントの共通クラス**：`.font-english { font-family: 'Philosopher', sans-serif; font-weight: normal; }` を作り、見出しやナビで使い回す。
- **ナビ装飾**：`.main-nav` を `display: flex`（横並び）＋`justify-content: center`＋`gap: 2.5rem`＋`font-size`＋`list-style: none`（黒丸を消す）。リンク色 `.main-nav a { color: #432; }`、`.main-nav a:hover { color: #0bd; }` でホバー色。

#### 4-6 キャッチコピーとカバー画像を作ろう
- **キャッチコピー**：`<header>` の下に `<h2 class="page-title font-english">We'll Make Your Day</h2>` を追加。`.page-title` に `font-size: 3rem`／`text-align: center`／`margin-top`／`line-height: 1.4`。
- **カバー画像（ファーストビュー）**：`<header>`＋`<h2>` を `<div class="cover cover-home">` で囲む。`.cover` は `background-size: cover`（縦横比保持で領域いっぱい）／`background-position: center bottom`／`height: 800px`。`.cover-home` に `background-image`（ホーム用）を指定。広げ方（`.cover`）と画像（`.cover-home`）を分けて他ページでも使い回す。
- COLUMN（うまく表示されない時のチェックリスト）：初学者は簡単なミスを見落としがち。詰まったら第8章「うまくいかない時の解決方法」を確認。

#### 4-7 コンテンツ部分を作ろう
- **見出しと文章**：`<section class="about">` に `<h3 class="heading-large font-english">About Cafe</h3>` と本文 `<p>`。`.about { max-width: 736px; padding: 0 1.5rem; margin: 3rem auto 0; }` で大画面でも中央にほどよい幅。
- **共通スタイルのまとめ**：`.page-title, .heading-large { font-size: 3rem; text-align: center; }` のように **カンマ区切り** で共通化し、共通でないものは別ルールで管理。
- **ボタン**：`<div class="align-center"><a class="btn" href="menu.html">メニューを見る</a></div>` で中央寄せ。`.btn` に `display: inline-block`（余白込みサイズを認識）／`background-color`／`color`／`border-radius: 8px`（角丸）／`padding`。`.btn:hover` で色変化。
- POINT（疑似クラス）：`:hover` は特定の状態（カーソルが乗っている時）だけスタイルを当てるセレクター。

#### 4-8 ボタンにアニメーションを加えよう
- **トランジション**：`transition` プロパティは指定時間をかけて始点→終点へプロパティを変化させる。`.btn` に `transition: .5s;` を足すと、`:hover` の背景色差分が0.5秒かけて「ふんわり」変化。`.5s` の先頭 `0` は省略可、`500ms` でも可。

| プロパティ | 意味 | 主な値 |
|---|---|---|
| `transition-property` | 適用するプロパティ | `all`（初期値）／プロパティ名／`none` |
| `transition-duration` | 実行にかかる所要時間（**必須**） | 数値 `s`（秒）／数値 `ms`（ミリ秒） |
| `transition-timing-function` | 速度・タイミング | `ease`（初期値）／`linear`／`ease-in`／`ease-out`／`ease-in-out`／`steps()`／`cubic-bezier()` |
| `transition-delay` | 始まるまでの待ち時間 | 数値 `s`／数値 `ms` |

- **まとめ記述**：`transition: background-color 1s ease-out 200ms;` のように「property duration timing-function delay」をスペース区切りで一括指定。項目は省略可だが **`transition-duration` は必須**（無いと動かない）。
- COLUMN：トランジションは始点・終点の2点間のみ。途中の別動作・繰り返し・自動再生は不可で、`:hover` など発動のきっかけが必要。

#### 4-9（前半）「footer」部分を作ろう ― 表と背景画像
- **表**：全体を `<footer class="page-footer">` で囲み、見出しに `heading-large`／`font-english` を付与。Information は `<table class="info">`（住所・電話・営業時間・店休日）で作成（2-10 の表の復習）。
- **表の装飾**：`.info` に `width: 100%`／`max-width: 544px`／`margin: auto`／`padding`／`border-spacing: 0`（セル間余白を消す）。`.info th, .info td { border-bottom: 1px solid #c9c2bc; }` で **行ごとに下線**。`th` は左寄せ・標準太さ・`padding`。
- **背景画像**：`.page-footer` に `background-image`＋`background-size: cover`（縦横比保持で画面いっぱい）＋`background-position: center`＋`padding-top: 12rem`。

#### 4-9（後半）「footer」部分を作ろう ― コピーライトと記号・特殊文字
- **概念**：Information の表の **下・`</footer>` の上** にコピーライトを追加してモバイル版ホームページを完成させる。
- **コピーライト**：`<div class="copyright">` 内に `<small>&copy; 2010 WCB Cafe</small>`。`<small>` は免責・著作権などの **注釈** を表すタグ。`&copy;` はブラウザーで **©** と表示させる記述。
- **CSS**：`.copyright` に `background-color`／`text-align: center`／`padding`／`margin-top`／`color` を指定。

| 記号 | 記述（文字参照） | 意味 |
|---|---|---|
| © | `&copy;` | コピーライト |
| ® | `&reg;` | 登録商標 |
| < | `&lt;` | 小なり |
| > | `&gt;` | 大なり |
| （空白） | `&nbsp;` | 半角スペース |

- COLUMN（記号・特殊文字）：環境によって化けやすい記号は「`&`（アンパサンド）で始まり `;`（セミコロン）で終わる」文字参照で表示させる。公式一覧は HTML Living Standard の Named character references を参照。

#### 4-10 レスポンシブに対応させよう
- **レスポンシブWebデザイン**：表示領域の **幅** によって見え方が変わるよう設計された Web サイト。掲載コンテンツは変更せず、**CSS だけ** でデバイスサイズに合わせて見た目を変える。
- **見た目を変える2つの工夫**：
  - **カラム数を減らす**：スマホでは横並びの複数カラムを **縦に積んで1カラム** にする（例：カルティエ）。
  - **ナビゲーションメニューの数を減らす**：狭い画面ではナビを一旦隠し、**メニューアイコン（2本線＝ハンバーガー）** のタップで展開（例：RENEUTO LAB）。
- **メディアクエリー（Media Queries）**：画面サイズに合わせて **適用する CSS を切り替える** 機能。`@media` で宣言し、丸括弧内に分岐点を指定。`min-width: 800px` は「最小の幅が 800px の時」＝ **800px 以上** に適用。
- **ブレークポイント**：CSS が切り替わる境目の画面サイズ。モバイルは縦持ちが多いため **縦にした時の幅（＝デバイスの短い方の幅）** を基準に考える。多くの場合 **小さい画面は 400px 付近／大きい画面は 800px 付近** を起点に設定。

| 適用先（デスクトップ版：`@media (min-width: 800px)` 内） | 指定内容 |
|---|---|
| `.page-title` / `.heading-large` / `.main-nav` | 文字サイズを拡大（`5rem` / `4rem` / `2rem`） |
| `.page-header` | `display: flex` ＋ `justify-content: space-between`（ロゴ左・ナビ右）＋ `padding-top: 1.5rem` |
| `.cover-home` / `.page-footer` | 背景画像を横長のデスクトップ用（`*-l.webp`）に差し替え、`padding` で余白調整 |
| `.info th` | `padding-left: 2.5rem` |

| 適用先（共通部分：メディアクエリーの外） | 指定内容 |
|---|---|
| `.wrapper`（`<header>` などに付与） | `max-width: 1120px`（最大幅）／`margin: auto`（中央寄せ）／`padding: 0 1.5rem`（左右余白） |

- **ポイント**：文字サイズ・ヘッダー横並び・背景差し替えは **ブレークポイント内**、`.wrapper`（最大幅・中央寄せ・左右余白）は画面幅に依らず効かせたいので **共通部分** に書く。

| iPad Pro (10.5") | 1112 | 834 |

#### 4-11 ファビコンを用意しよう
- **ファビコンとは**：ブックマーク・タブでサイト名の横に表示される小さなアイコン。**Favorite ＋ Icon**。実寸は 16px 四方ほどと小さいが、複数タブやブックマークからサイトを区別するのに役立つ。
- **デザインの考え方**：多くはロゴマークを縮小・簡略化したもの。小さくサイズ制限がある分、**テーマカラーやシンプルな図形** でわかりやすくするのがポイント（例：YouTube は再生ボタンのみ、任天堂はゲームキャラのドット絵）。
- **画像の用意**：Illustrator／Photoshop 等で作成。**縦横 32px 以上の正方形・PNG 形式**（生成サービス利用時は 260px 以上推奨）。WCB Cafe はロゴが複雑なため簡略版をファビコンに設定。
- **読み込み**：作成画像を `favicon.png` として `images/` に保存し、`<head>` 内に `<link rel="icon" type="image/png" href="images/favicon.png">` を記述。
- **生成サービス**：`RealFaviconGenerator`（https://realfavicongenerator.net/ ）に大きめ画像を渡すと、デスクトップ用だけでなくスマホのホームスクリーンや Windows のスタートメニュー用まで一括生成し、表示プレビューやエラーも確認できる。

#### 4-12 ボタンアニメーションのカスタマイズ例
- **概要**：Chapter 4 のホームページに手を加えるデザインアレンジ集。いずれも `.btn` の `transition`（例 `.5s`）と `:hover` を組み合わせ、`:hover` 側のプロパティだけを差し替える。

| プロパティ | 役割 | 値の書き方 | 例 |
|---|---|---|---|
| `scale` | 拡大 | 値1つ＝上下左右同率／「横 縦」の2値も可。**単位不要** | `scale: 1.2;` ／ `scale: 1.2 1.5;`（横1.2・縦1.5倍） |
| `rotate` | 回転 | 単位 `deg`。`360deg` で1回転。マイナスで反時計回り | `rotate: -5deg;` |
| `translate` | 移動 | 値1つ＝横方向／「横 縦」の2値も可。正＝右, 負＝左 | `translate: 10px;` ／ `translate: 10px -20px;`（右斜め上） |

### 第5章 2カラムのWebサイトを制作する

> 「WCB CAFE」に2カラムの「News」ページを追加する章。メインエリア（記事）とサイドバー（Category／Archives）の2列レイアウトを、モバイル→デスクトップの順に作る。

#### 5-1 2カラムのレイアウトとは
- **概念**：**カラム**＝縦の列。垂直方向にコンテンツを区切って並べるのが**カラムレイアウト**。News ページは2列＝**2カラム**で作る。
- **構成要素**：ページ上部の画像 `<div>`／見出し `<h2>`／メインエリア `<article>`／サイドバー `<aside>`。
- **特徴**：コンテンツ量の多いニュース・ブログ向き。モバイルでは縦積み、デスクトップで横並び。幅比率は **メイン:サイド＝2:1／3:1** が一般的。画面を半分で割るものは**スプリットスクリーン**。

#### 5-2 2カラムページの制作の流れ
- ホーム同様**モバイルファースト**で作成。流れは ①ページ全体の見出し → ②メインエリア → ③サイドバー（カテゴリー一覧・アーカイブ）→ ④レスポンシブ対応（デスクトップで2カラム化）。
- **POINT**：カラム間の**余白**をしっかり取り、窮屈な印象を避ける。

#### 5-3 ページ全体の見出しを作成しよう
- **ファイル準備**：ロゴ・ナビ・フッターは `index.html` と共通なので、VS Code で `index.html` を右クリック→コピー→貼り付けし、**`news.html`** にリネーム。
- **HTML編集**：`<title>` を **`WCB Cafe - NEWS`** に変更（description も調整）／ホーム固有の `<section class="about">` を**削除**。
- **クラス名・見出し変更**：ヘッダーを囲む `<div>` を **`cover cover-home` → `sub-cover cover-news`** に変更／見出しテキストを **「News」** に変更。
- **CSS装飾**：`style.css` に **`.sub-cover`**（高さ・下余白）と **`.cover-news`**（News 用背景画像）を追記。

#### 5-4 メインエリアを作ろう
- **HTML**：全体を **`<main class="wrapper">`** で囲み、`<article class="post">` → `<header>` に記事タイトル `<h2 class="post-title">`、カテゴリー `<ul class="post-cat">`（複数指定前提）、日付 `<time class="post-date" datetime="2024-09-28">` を配置。
- **`<time>` の datetime 属性**：`年-月-日` 形式。表示テキストと機械可読値を分けられ、検索エンジンに拾われ**検索結果に反映**（SEOに有効）。
- **CSS装飾**：`.post-info` を `display: flex`＋`gap` で横並び／`.post-cat` を `display: flex`＋`list-style: none`、`<a>` に背景色・`border-radius`・余白でボタン風に／`.post-thumbnail`・`.post p` に下余白。
- **COLUMN**：代表的なリセットCSS＝**`normalize.css`／`ress.css`**。

#### 5-5 サイドバーを作ろう
- **HTML**：`<article>` の下に **`<aside class="sidebar">`**。`Category`・`Archives` の見出し `<h3 class="heading-medium font-english">` と、`<ul class="side-menu">` のリンクリストを配置。
- **見出しの共通化**：`.page-title, .heading-large, .heading-medium { … }` と**カンマ区切り**でまとめて適用（デスクトップで個別サイズ変更するため別クラスにしておく）。
- **サイドバー装飾**：`.side-menu` を `list-style: none`（丸マーク非表示）＋中央揃え／`.side-menu li` に下線／`.side-menu a` に **`display: block`**（リスト幅いっぱいをクリック可能に）＋余白＋ホバー色。

#### 5-6 レスポンシブに対応させよう
- `@media (min-width: 800px)` 内で見出し・記事タイトルを拡大、ヘッダーを Flexbox 化、`.sub-cover` を低く控えめに、背景画像を**デスクトップ用 `-l.webp`** に差し替え。
- **2カラム化**：`<main>` に **`news-contents`** クラスを追加し、`display: flex`＋`justify-content: space-between`。幅は **`%` 指定**（メイン70%／サイド22%／余白8%）で、画面幅が変わっても伸縮。

#### 5-7 2カラムページのカスタマイズ例（3カラム化）
- **3カラム化**：`news-contents` 内に **`<aside class="ad">`**＋バナー画像を追加。メイン幅を 70%→60% に調整。
- **表示順の変更**：**`order` プロパティ**（`display: flex` 内で有効）で「バナー→メイン→サイド」に並べ替え（`order: 1/2/3`）。
- **COLUMN**：**重要なコンテンツ（記事）は HTML の上部**に書き、見た目の順序は CSS の `order` で調整（読み込み順・SEO配慮）。

### 第6章 タイル型のWebサイトを制作する

> 「WCB CAFE」に「Menu」ページを追加する章。9つのメニュー画像を**CSSグリッド**で敷き詰める**タイル型レイアウト**を、モバイル2列→デスクトップ3列で作り、一部を大きく見せる配置まで行う。

#### 6-1 タイル型レイアウトとは
- **概念**：画像やテキストを敷き詰めて一度に見せるレイアウト。ショッピングサイトの商品一覧・ギャラリー・Webデザインギャラリーなどでよく使われる。
- **特徴**：各ボックスの大きさや高さに変化をつけたり、左右のラインを揃えることで、画像サイズが違ってもきれいに見せられる。

#### 6-2 タイル型ページの制作の流れ
- 上部は背景画像＋ページタイトル（News と同構成）。**CSSグリッド**で9つのメニューをタイル型に並べ、モバイル→デスクトップの順で作成。
- レスポンシブは**CSSグリッドの応用**（列数・`gap` の変更）で、画面幅に合わせて表示を切り替える。

#### 6-3 ページ全体の見出しを作成しよう
- `index.html` を複製して **`menu.html`** を作成。`<title>` を **`WCB Cafe - MENU`** に変更。
- News 固有の **`<main class="wrapper news-contents">` を削除**。ヘッダーの `<div>` を **`sub-cover cover-menu`** に変更、見出しを **「Menu」** に。
- CSS に **`.cover-menu`**（モバイル用背景 `cover-menu-s.webp`）を追記。

#### 6-4 タイル型レイアウトを設定しよう
- **HTML**：`<div class="sub-cover cover-menu">` と `<footer>` の間に **`<main class="wrapper grid">`** を置き、中に **`<div class="item">`（`<img>`＋`<p>`）を9個**（`menu1.webp`〜`menu9.webp`）。
- **CSSグリッド**：`.grid` に `display: grid`／`gap`（縦 横）／**`grid-template-columns: repeat(2, 1fr)`**（モバイル2列）。`1fr` は伸縮単位、`repeat(回数, 幅)` でまとめて指定。
- **画像**：`.item img` に **`aspect-ratio: 1/1`**（正方形）＋**`object-fit: cover`**（はみ出しを自動トリミング＝歪み防止）＋`border-radius`。

#### 6-5 背景画像の変更・要素を3つずつ並べよう
- `@media (min-width: 800px)` 内で `.cover-menu` を**デスクトップ用 `cover-menu-l.webp`** に差し替え。
- **列数変更**：`grid-template-columns` を `repeat(2, 1fr)` → **`repeat(3, 1fr)`**（2列→3列）。`gap` を広げ、文字サイズも拡大。

#### 6-6 サイズの違う要素を配置しよう
- **HTML**：大きく見せたい要素に **`item-big`** クラスを追加（例：最初のメニュー）。
- **範囲指定**：**`grid-column`**（横）／**`grid-row`**（縦）を**グリッドライン番号**で「始まり/終わり」を指定（左端・上端が1番）。`grid-column: 1/3; grid-row: 1/3;` で2×2マスに拡大。
- **高さ調整**：`.item-big img { height: 94.5%; width: 100%; }` で大きい画像の高さを揃え、整った見た目に。

### 第7章 外部メディアを利用する

> 「WCB CAFE」に「Contact」ページを追加する章。フォームの代わりに **mailto リンク**を使い、**Googleマップ・Instagram・X・YouTube** などの外部メディアを埋め込む。仕上げにレスポンシブ2カラム化・**OGP設定**・**Googleマイマップ**のカスタマイズまで行い、Contact ページと本サイトを完成させる。

#### 7-1 お問い合わせページの制作の流れ
- **概念**：CONTACT ページはフォームの装飾に加え、**Googleマップ**や各種 **SNS（Instagram／X／YouTube）のプラグイン**を表示する。
- **構成**：上部＝背景画像＋ページタイトル／中段＝お店の情報（Information）と地図・メール宛ボタン／下部＝SNS・動画を横並び（モバイルは縦並び）。
- **前提**：本章は Google／Facebook／X／YouTube のアカウントが**登録済み**として進める。

#### 7-2 ページ全体の見出しを作成しよう
- **ファイル準備**：`menu.html` を複製して **`contact.html`** にリネーム。
- **HTML編集**：`<title>` を **`WCB Cafe - CONTACT`** に変更／Menu 固有の `<main class="wrapper grid">`（`item`×9）を**削除**／ヘッダーの `<div>` を **`cover-menu` → `cover-contact`** に変更／見出しを **「Contact」** に。
- **CSS装飾**：`style.css` に **`.cover-contact`**（CONTACT 用背景画像）を追記。

#### 7-3 地図を表示しよう
- **お店の情報**：`<main class="wrapper">` 内に **`<section class="location">`**。中に `<div class="location-info">`（`<table class="info">`：住所／電話／営業時間／店休日）と `<div class="location-map">`。情報の表は**フッターの `table.info` をコピペ流用**。
- **余白調整**：`.location`／`.location-info`／`.info` に `margin`。
- **Googleマップ埋め込み**：Googleマップで住所検索→[共有]→[地図を埋め込む]→サイズ設定→[HTMLをコピー]で `<iframe>` を取得し貼り付け（**埋め込み＝利用規約に同意**）。
- **見た目調整**：`iframe { width: 100%; }` ではみ出しを枠内におさめる。

#### 7-4 メールアドレス宛のリンクを用意しよう
- **概念**：フォームは実装のハードルが高いため、手軽な **mailto リンク**で問い合わせ導線を作る。
- **HTML**：`<section class="location">` の下に **`<section class="email">`**（見出し・メッセージ・ボタン）。ボタンの `<a>` の `href` に **`mailto:アドレス`**（**半角スペースを入れず詰めて記述**）。
- **CSS装飾**：`.email` に背景色・角丸・余白と **`max-width`**（デスクトップで広がりすぎ防止）。

#### 7-5 Instagramの投稿を挿入しよう
- **SNSレイアウト**：`<section class="email">` の下に **`<section class="sns">`**、各SNS用に **`<div class="sns-item">`**（Instagram／X／YouTube）。仮テキストを入れておき、`.sns-item` に下余白。
- **Instagram取得**：投稿の […] →[埋め込み]→**「キャプションを追加」のチェックを外す**→[埋め込みコードをコピー]→貼り付け。
- **表示修正**：枠は出るが写真が出ない→`<script src="//…embed.js">` に **`https:` を追加**して解決（プロトコル相対URLの問題）。

#### 7-6 Xプラグインを挿入しよう
- **コード取得**：X の生成ページ（publish.twitter.com）で **X の URL（twitter.com/アカウント名）** を入力→[enter]→**「Embedded Timeline」**を選択→[set customization options]で幅・高さ調整→[Copy Code]。
- **貼り付け**：仮の「Xの投稿」部分に貼る。**`<a class="twitter-timeline">`＋`widgets.js`** の構成。※設定は英語のみ、埋め込みで Developer Agreement に同意。

#### 7-7 YouTube動画を挿入しよう
- **コード取得**：YouTube で動画下の[共有]→[埋め込む]→コードをコピーして貼り付け（設定不要で最も手軽）。※[開始位置]で再生開始時間を指定可。
- **比率調整**：貼ったままだと左右が見切れるため、`.sns-youtube` に **`aspect-ratio: 16/9`**（YouTube 推奨比率）＋iframe の高さを **`100%`** に指定して縦横比を正しく反映。

#### 7-8 レスポンシブに対応させよう
- `@media (min-width: 800px)` 内で背景画像を**デスクトップ用（`cover-contact-l.webp`／`footer-l.webp`）**に差し替え。
- **情報と地図を横並び**：`.location` に `display: flex`、`.location-info` 32%／`.location-map` 64%。
- **SNSを横並び**：`.sns` に `display: flex`。Flexは子の高さを引き伸ばすため、**`align-items: flex-start`**（上揃え）で YouTube が縦長になるのを防ぐ。

#### 7-9 OGPの設定をしよう
- **概念**：**OGP（Open Graph Protocol）**＝SNS シェア時にタイトル・説明文・画像を制御する仕組み。設定なし＝URLとタイトルのみ、設定あり＝タイトル・説明文・画像が表示され、見え方が大きく変わる。
- **HTML**：`<head>` 内に **`<meta property="og:url／og:type／og:title／og:description／og:image">`** を記述。`og:type` は website／article。`og:image` は Facebook 推奨 **1200×630px**。
- **確認ツール**：Meta（Facebook）の**シェアデバッガー**、X の **Card validator** で事前確認。

#### 7-10 Googleマイマップを利用しよう（地図のカスタマイズ）
- **概念**：標準の埋め込み地図よりカスタマイズしたい場合、**Googleマイマップ**でオリジナル地図を作る。※節タイトルの正確な文言は写真から一部判読困難（※未確認）。
- **作成**：[新しい地図を作成]→タイトル・説明を編集→[保存]。住所を検索→吹き出しの[地図に追加]で確定。
- **カスタマイズ**：[スタイル]でピンの**色・種類・カスタムアイコン（画像アップロード）**、地図の背景スタイル（航空写真・地形・モノクロ等）を変更。
- **公開・埋め込み**：初期は**非公開**のため[共有]で公開範囲を変更→タイトル右メニュー→[自分のサイトに埋め込む]で `<iframe>`（**`maps/d/embed?mid=…`**）を取得し貼り付け。width/height は表示サイズに合わせて修正。

### ファイルの種類と拡張子（覚える）

| 種類 | 役割 | 拡張子 |
|---|---|---|
| HTMLファイル | コンテンツ（文章・画像）を記述 | `.html` |
| CSSファイル | 装飾（色・文字サイズ・配置）を記述 | `.css` |
| 画像（JPG） | 写真・色数の多い画像向き | `.jpg` |
| 画像（PNG） | ロゴ・色数の少ない画像向き。背景透過可 | `.png` |
| 画像（GIF） | 256色まで。単純なイラスト・アニメ向き、透過可 | `.gif` |
| 画像（SVG） | ベクター形式。拡大縮小で劣化しない | `.svg` |
| 画像（WebP） | 軽量。古いブラウザー（IE等）は非対応 | `.webp` |

- ファイル名は **すべて半角英数字**（全角・日本語名は不可。表示されない原因になる）。
