# 今日学んだこと
### 2026-06-28〜07-01｜第4章 シングルカラムのWebサイト制作（続き）4-5後半〜4-12（ロゴ/ナビ仕上げ・キャッチコピー/カバー画像・コンテンツ/ボタン・トランジション・footer・レスポンシブ・ファビコン・ボタンアニメ応用）

**何をしたか**
- **4-5（続き）** ロゴの横幅（`.logo { width: 210px; }`）を調整し、欧文フォント Philosopher を使い回す共通クラス `.font-english` を作成。ナビ `.main-nav` を `display: flex` で横並びにし、`gap`・文字サイズ・`list-style: none`、リンク色と `a:hover` の色を指定してヘッダーを仕上げました。
- **4-6 キャッチコピーとカバー画像を作ろう**：`<header>` の下にキャッチコピー `<h2 class="page-title font-english">We'll Make Your Day</h2>` を追加。`<header>`＋`<h2>` を `<div class="cover cover-home">` で囲み、`background-size: cover` で **画面いっぱいの背景画像（ファーストビュー）** を設置しました。

- **4-8 ボタンにアニメーションを加えよう**：**トランジション**（`transition` プロパティ）で `:hover` 時に背景色をふんわり変化させました。`transition` 関連の各プロパティ（`-property`／`-duration`／`-timing-function`／`-delay`）と、まとめ記述（値をスペース区切り）を学びました。
- **4-9（前半）「footer」部分を作ろう**：`<footer class="page-footer">` に Information の表（`<table class="info">`）を設置し、`border-spacing: 0` と `border-bottom` で **行ごとの下線** に。`background-size: cover` でフッターにも背景画像を設置しました。
- **4-9（後半）**：Information 表の下・`</footer>` の上に **コピーライト**（`<small>&copy; 2010 WCB Cafe</small>`）を追加し、**モバイルサイズのホームページが完成**。COLUMN で **記号・特殊文字**（文字参照）を学びました。
- **4-10 レスポンシブに対応させよう**：**メディアクエリー**（`@media (min-width: 800px)`）・**ブレークポイント**、デスクトップ版の文字拡大・Flexbox ヘッダー・`.wrapper` 最大幅・背景画像差し替えを学びました。
- **4-11 ファビコンを用意しよう** ／ **4-12 ボタンアニメーションのカスタマイズ例（`scale`／`rotate`／`translate`）** まで進めました。

**学んだ用語**

| 用語 | やさしい説明 |
|---|---|
| `.font-english`（共通クラス） | 欧文フォント `Philosopher` を使い回すための自作クラス（`font-family`・`font-weight: normal`） |
| `display: flex`（`.main-nav`） | ナビの `<li>` を **横並び** にする起点（Flexbox）。3-16 の復習 |
| `gap`（Flexbox） | フレックスアイテム同士の **間隔**（例：`2.5rem`）。ここではナビ項目の間隔に使用 |
| `list-style: none` | リストの先頭マーカー（黒丸）を消す指定。ナビを見た目上ただの横並びにする |
| `a:hover`（疑似クラス） | リンクにカーソルを合わせた時に適用されるスタイル |
| 疑似クラス | 特定の **状態** にある要素にスタイルを当てるセレクター（例：`:hover`＝カーソルが乗っている時のみ適用） |
| キャッチコピー（`.page-title`） | ページ上部中央に大きく置く決め文句。`<h2>` で表す |
| カバー画像 / `.cover` / `.cover-home` | ページ上部を覆う大きな背景画像。`.cover`（共通の広げ方）と `.cover-home`（ホーム固有の画像）を分けて管理 |
| `background-size: cover` | 画像の **縦横比を保ったまま** 領域いっぱいに広げる（はみ出しは切れる）。3-10 の復習 |
| `background-position` | 背景画像の表示開始位置（例：`center bottom`／`center`） |
| ファーストビュー | Web ページにアクセスした時、**最初に表示される** 画面領域 |
| `<section class="about">` | 1つのテーマ（お店紹介）を持つコンテンツのまとまり |
| `.heading-large`（共通見出しクラス） | 大きめ見出しの共通スタイル（文字サイズ・中央揃え・下余白） |
| カンマ区切りでの共通化 | `.page-title, .heading-large { … }` のように **共通スタイルをまとめて** 管理する書き方 |
| `<a class="btn">` | リンクをボタンとして見せる。中央寄せは `.align-center` の `<div>` で囲む |
| `display: inline-block` | ボタンに **余白（padding）を含めたサイズ** を認識させ、中央寄せや大きさ指定を効かせるための指定 |
| `border-radius` | 四角形の **角を丸める** プロパティ（例：`8px`） |
| `transition`（トランジション） | 指定した **時間をかけて** プロパティを始点→終点へ変化させる。単純な動きに使う |
| `transition-property` | 変化させる対象プロパティ（`all`＝全部／プロパティ名／`none`） |
| `transition-duration` | 変化にかかる **所要時間**（`s`＝秒／`ms`＝ミリ秒）。**省略不可**（無いと実行されない） |
| `transition-timing-function` | 変化の **速度・タイミング**（`ease`＝初期値／`linear`／`ease-in`／`ease-out`／`ease-in-out`／`steps()`／`cubic-bezier()`） |
| `transition-delay` | 変化が **始まるまでの待ち時間**（`s`／`ms`） |
| まとめ記述 `transition: 〜` | `property duration timing-function delay` をスペース区切りで一括指定（例：`background-color 1s ease-out 200ms`） |
| `border-spacing: 0`（table） | 表の **セル間の余白をなくす**。行ごとに1本線を引く表現の下地になる |
| `border-bottom`（`.info th, td`） | セルの **下線**（例：`1px solid #c9c2bc`）。行ごとの区切り線に |

**つまずき・気づき**
- 欧文フォントや中央揃えなど **複数箇所で使う装飾は共通クラス** にしておく（`.font-english`・`.align-center`）。さらに 4-7 では `.page-title, .heading-large { … }` のように **カンマで共通スタイルをまとめる** と管理が楽（共通でないものは別ルールに分ける）。
- ボタンは `<a>`（インライン要素）なので、そのままだと `padding` や中央寄せが効きにくい。`display: inline-block` を付けて **余白込みのサイズを認識** させるのがポイント。
- カバー／フッターの背景画像は `background-size: cover` で **縦横比を保ったまま領域いっぱい**（はみ出しは切れる）。`.cover`（広げ方＝共通）と `.cover-home`／画像指定を **クラス分け** して他ページで使い回す設計。
- **トランジションは始点・終点の2点間のみ**。途中の別動作・繰り返し・自動再生はできず、`:hover` など発動のきっかけが必要（複雑な動きは別のアニメーション手法）。まとめ記述 `transition: 〜` では **`transition-duration`（時間）が必須**（無いと動かない）。`.5s` の先頭 `0` は省略可、`500ms` でも可。
- `transition-timing-function` の既定は `ease`（開始・終了が緩やか）。`ease-in`＝最初ゆっくり、`ease-out`＝最初速く、と挙動が異なる。
- 表の行区切り線は、`border-spacing: 0`（セル間余白を消す）＋`th`/`td` の `border-bottom` で「行ごとに1本線」を表現する。`<table border="1">` のような枠線ではなく **CSS で装飾** するのが実践的（2-10 の復習＋発展）。
- `.about`（本文）も `.info`（表）も `max-width` を指定しておくと、**大画面で幅が広がりすぎず** 中央にほどよく収まる。モバイルでは見た目が変わらないのがポイント。
