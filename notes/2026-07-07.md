**学んだ用語**

| 用語 | やさしい説明 |
|---|---|
| 外部メディア | Googleマップ・Instagram・X・YouTube など、他サービスが提供する機能を自分のページに埋め込んで使うもの |
| プラグイン | 各サービスが発行する埋め込み用コード。貼り付けると地図や投稿がページ内に表示される |
| `cover-contact` | ホーム／News／Menu と同様、CONTACT ページ固有の背景画像を割り当てるクラス（`sub-cover` と組で使用） |
| `<section class="location">` | 店舗情報と地図をまとめる領域 |
| `location-info` / `location-map` | 店舗情報（表）を入れる領域と、地図（iframe）を入れる領域 |
| `<table class="info">` | 住所・電話・営業時間・店休日を並べる表。フッターの同名テーブルを流用できる |
| `<iframe>` | 別ページ（ここではGoogleマップ）を枠として埋め込むタグ。幅は `100%` に調整 |
| mailto リンク | `href="mailto:アドレス"` でメールアプリを起動するリンク。プログラム不要で手軽に問い合わせ導線を作れる |
| `<section class="email">` | お問い合わせメッセージとボタンを入れる領域 |
| `.btn`（`align-center` 内の `<a>`） | ボタン風リンク。お問い合わせ導線に使用（既存クラスを流用） |
| `<section class="sns">` / `sns-item` | SNS 全体を囲む領域と、各 SNS（Instagram／X／YouTube）1枠ぶんのクラス |
| `blockquote class="instagram-media"` | Instagram の埋め込み本体。`data-instgrm-permalink` に投稿URLが入る |
| `embed.js` | Instagram 埋め込みを描画する外部スクリプト。`//` のままだと写真が出ず、`https:` を足すと表示される |

**できるようになったこと**：`menu.html` を複製して `contact.html` を用意し、`title` を「WCB Cafe - CONTACT」に変更、Menu 固有コンテンツを削除、クラス名を `cover-contact` に変更、見出しを「Contact」化するまでの一連の準備。`<section class="location">`＋`location-info`（`<table class="info">`）／`location-map` による店舗情報エリアの構築と、フッターの表の流用。Googleマップの埋め込みコード（`<iframe>`）取得・貼り付けと、`iframe { width: 100%; }` でのはみ出し調整。フォームの代わりに `href="mailto:…"` でお問い合わせ導線を用意し、`.email`（`max-width`／背景色／角丸／余白）でメッセージ領域を装飾。`<section class="sns">`＋`sns-item` の SNS レイアウト作成と、Instagram 埋め込みコードの取得・貼り付け、`embed.js` の `https:` 追加による表示不具合の修正。

**つまずき・気づき**
- Googleマップも SNS も、公式が発行する **埋め込みコードをコピー＆ペーストするだけ** で設置できる。ただし `<iframe>` は幅を指定しないとレイアウトからはみ出すので、`width: 100%` の調整が必要。
- Instagram 埋め込みで **枠は出るのに写真が表示されない** のは、貼り付けたコードの `<script src="//…embed.js">` が「プロトコル相対URL（`//` 始まり）」になっているため。先頭に `https:` を足すと解決する。埋め込みは「貼るだけ」で終わらず、一部修正が要る場合がある。
- 店舗情報の表は、フッターの `<table class="info">` を **コピペで再利用** できた。第6章で当てたスタイルもそのまま引き継げるので、`margin` などの微調整だけで済む。
- フォームは設置にプログラミングが必要でハードルが高いが、**mailto リンク** なら `href` を書くだけでお問い合わせ導線を作れる。`mailto:` は **半角スペースなどを入れず詰めて** 書かないと正しく動作しない。

### 2026-07-07（続き）｜第7章 7-6〜7-10（X／YouTube 埋め込み／レスポンシブ2カラム化／OGP／Googleマイマップ）

**何をしたか**
- **7-6 Xプラグインを挿入しよう**：X（旧Twitter）の投稿を表示するプラグインを挿入します。生成ページ（https://publish.twitter.com/）を開き、「Enter a Twitter URL」に **X の URL（twitter.com/アカウント名）** を入力して［enter］。表示された「Here are your display options」で **「Embedded Timeline」** をクリック。下にコードが出るので、サイズを変えたい場合は **［set customization options］** で幅・高さを設定し、**［Copy Code］** でコピー。7-5 で仮に「Xの投稿」と書いておいた場所に貼り付けました（c7-06-1）。※設定ページは英語のみ。埋め込みで Developer Agreement / Policy に同意したことになります。
- **7-7 YouTube動画を挿入しよう**：世界最大の動画共有サービス YouTube の動画を表示します。他のプラグインと比べて設定不要で簡単。YouTube（https://www.youtube.com/）で掲載したい動画を開き、動画下の **［共有］→［埋め込む］** をクリック → 表示された「動画の埋め込み」コードをコピーし、仮に「YouTube動画」と書いておいた場所に貼り付け（c7-07-1）。※再生開始位置を指定したい場合は［開始位置］にチェックを入れて時間を指定できます。
- **7-7（続き）YouTube動画の表示比率を調整する**：貼り付けたままだと左右が若干見切れて比率が保たれないため、CSS で **`aspect-ratio: 16/9`**（YouTube 推奨比率）を追加し、**iframe の高さを `100%`** に指定して縦横比を正しく反映させました（c7-07-2）。
- **7-8 レスポンシブに対応させよう**：SNS までできたので、最後にデスクトップ対応。メディアクエリー（`@media (min-width: 800px)`）内で、**背景画像をデスクトップ用（`cover-contact-l.webp` / `footer-l.webp`）に差し替え**。さらに **お店の情報と地図を横並び**（`.location` に `display: flex`、`.location-info` を幅 32%、`.location-map` を 64%）に、**各 SNS も横並び**（`.sns` に `display: flex`）にしました（c7-08-1〜2 付近）。
- **7-8（続き）SNS の縦揃えを調整**：`display: flex` は子要素の高さを揃えようと中身を引き伸ばす性質があり、そのままだと YouTube 動画が縦長になってしまう。そこで **`.sns` に `align-items: flex-start`** を加えて上揃えにし、余白を調整して完成させました。
- **7-9 OGPの設定をしよう**：**OGP（Open Graph Protocol）** とは、SNS で Web サイトがシェアされた時にタイトル・説明文・画像を意図通りに表示させる仕組み。設定がないと URL とタイトルのみ、設定があるとページタイトル・説明文・画像が表示され、見え方が大きく変わります。HTML の `<head>` 内に `og:url`／`og:type`／`og:title`／`og:description`／`og:image` を **`<meta property>` で記述**（c7-09 付近）。※og:image は Facebook 推奨サイズ 1200×630px の画像を用意。設定確認は Meta（Facebook）の「シェアデバッガー」や X の「Card validator」で行います。
- **7-10 Googleマイマップを利用しよう（地図のカスタマイズ）**※節タイトルの正確な文言は写真から一部判読できず（※未確認）：標準の埋め込み地図よりカスタマイズ性を上げたい場合、**Googleマイマップ**（https://www.google.com/maps/）で ［新しい地図を作成］→ タイトル・説明を編集して［保存］。住所を検索して吹き出しの ［地図に追加］ で場所を確定。［スタイル］アイコンからピンの **色・種類・カスタムアイコン（画像アップロード）** を変更でき、地図の背景スタイル（航空写真・地形・モノクロ等）も選べます。作成した地図は初期状態が非公開のため、［共有］で **「リンクを知っている全員が表示可」＋「一般公開」** に変更。最後にタイトル右のメニュー →［自分のサイトに埋め込む］で `<iframe>` コードを取得し、`contact.html` の地図表示部分に貼り付けて完成（c7-10-1）。埋め込みコードの幅（width）・高さ（height）は表示したいサイズに合わせて修正します。

| メディアクエリー `@media (min-width: 800px)` | 画面幅 800px 以上（デスクトップ）にのみ適用する CSS の書き方 |
| OGP（Open Graph Protocol） | SNS シェア時にタイトル・説明文・画像を制御する `<meta>` 群 |
| `og:url` / `og:type` / `og:title` / `og:description` / `og:image` | それぞれ URL／種類(website・article)／タイトル／説明文／表示画像を指定 |
| シェアデバッガー / Card validator | OGP・X カードの表示を事前確認する公式ツール（Meta／X） |
| Googleマイマップ | ピンやアイコン、色をカスタマイズしたオリジナル地図を作れる Google のサービス |
| `maps/d/embed?mid=…` | マイマップ埋め込み iframe の URL。`mid` が作成した地図の ID |

**学んだ主な構文**

**できるようになったこと**：X（Embedded Timeline）を publish.twitter.com で生成して埋め込み、YouTube 動画を［共有］→［埋め込む］で取得して挿入するまで。YouTube の見切れを `aspect-ratio: 16/9`＋iframe 高さ 100% で補正。メディアクエリー（`@media (min-width: 800px)`）でデスクトップ用背景画像に差し替え、`.location` の `display: flex`（情報32%／地図64%）と `.sns` の `display: flex`＋`align-items: flex-start` による横並び2カラム化。`<head>` への OGP（`og:url`／`og:type`／`og:title`／`og:description`／`og:image`）記述と、シェアデバッガー／Card validator での確認。Googleマイマップでピン・アイコン・地図スタイルをカスタマイズし、公開設定にして `maps/d/embed?mid=…` の iframe を埋め込むところまで。これで CONTACT ページと第7章が完成。

**つまずき・気づき**
- Flexbox で横並びにすると、子要素は **高さを揃えようと引き伸ばされる**。YouTube 枠が縦長になるのは `align-items: flex-start`（上揃え）で防げる。伸縮の性質を理解しておくとレイアウト崩れの原因が読める。
- YouTube 動画は貼り付けたままだと左右が見切れる。**`aspect-ratio: 16/9`＋iframe の高さ指定** で本来の比率を保てる。`aspect-ratio` は比率指定の定番プロパティ。
- OGP は **「設定なし＝URLとタイトルのみ」「設定あり＝タイトル・説明文・画像」** と、SNS シェア時の見え方が大きく変わる。集客を意識するなら `og:image`（Facebook 推奨 1200×630px）まで用意する。実際の表示は投稿前に **公式の確認ツール** で検証できる。
- 埋め込み地図は 2 段階ある：手軽な **Googleマップの iframe**（7-3）と、アイコン・色まで作り込める **Googleマイマップ**（7-10）。後者は初期状態が非公開なので、**公開設定に変更しないと他人に表示されない** 点に注意。
- 背景画像はモバイル用（`-s.webp`）とデスクトップ用（`-l.webp`）を **メディアクエリーで出し分ける** のが、これまでのページと共通の設計パターン。
