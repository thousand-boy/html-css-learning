# 2026-05-26｜条件分岐（if・else・else if・比較演算子・論理演算子・switch文）

---

**何をしたか**
- Progateで **条件分岐** を学びました。
- 「もし〇〇なら●●する」という処理の書き方（if・else・else if）と、条件を比較するための演算子（比較演算子・等価演算子・厳密等価演算子）を学びました。
- 複数の条件を組み合わせる論理演算子（`&&` / `||`）と、値によって処理を振り分ける `switch` 文を実践しました。

---

**学んだ用語**

| 用語 | やさしい説明 |
|---|---|
| 真偽値（boolean） | `true`（正しい）か `false`（正しくない）の2種類だけの値。条件式を評価すると必ずどちらかになる |
| 比較演算子 | 2つの値の大小を比べる記号（`<` / `<=` / `>` / `>=`） |
| 等価演算子 `==` | 左右の値が等しければ `true`。型変換が行われるため `12 == "12"` も `true` になる |
| 厳密等価演算子 `===` | 型変換なしで厳密に比較。`12 === "12"` は `false`。原則こちらを使う |
| `!==` | 厳密に異なるとき `true`。型変換なし |
| if文 | 「もし条件が `true` なら処理を実行する」という構文 |
| else | if文の条件が `false` のときに実行する処理を書く |
| else if | ifとelseの間に追加できる条件分岐。上から順に評価し、最初に `true` のブロックだけ実行される |
| `&&`（AND） | 「かつ」。すべての条件が `true` のとき `true` になる |
| `\|\|`（OR） | 「または」。1つでも `true` なら `true` になる |
| switch文 | 1つの値に対して複数のケース（`case`）で処理を分岐させる構文。`switch(値) { }` と書く（末尾セミコロン不要） |
| `case` | switch文の中で「この値のとき」を指定するキーワード |
| `break;` | switch文を終了する命令。**忘れると一致したcase以降も続けて実行されてしまう**（フォールスルー） |
| フォールスルー | `break;` がないときに発生する現象。一致したcaseの次のcaseも実行され続けてしまう |
| `default:` | どの `case` にも当てはまらなかった場合の処理。if文の `else` に相当 |

---

**学んだ主な構文**

```javascript
// 真偽値の確認
console.log(12 > 10);  // true
console.log(12 > 20);  // false

// if / else / else if
if (age >= 20) {
  console.log("20歳以上です");
} else if (age >= 10) {
  console.log("20歳未満ですが10歳以上です");
} else {
  console.log("10歳未満です");
}

// 厳密等価演算子
if (password === "ninjawanko") {
  console.log("ログインに成功しました");
}

// 論理演算子（&&：かつ）
if (age >= 20 && age < 30) {
  console.log("20代です");
}

// switch文（break必須・defaultで漏れを防ぐ）
const color = "赤";
switch (color) {
  case "赤":
    console.log("ストップ！");
    break;
  case "黄":
    console.log("要注意");
    break;
  case "緑":
    console.log("進め！");
    break;
  default:
    console.log("colorの値が正しくありません");
    break;
}
```

---

**できるようになったこと**
- 条件式の結果（`true` / `false`）を理解し、if文で処理を分岐させることができる。
- `===` と `==` の違いを説明でき、厳密等価演算子を適切に使い分けられる。
- `&&`（かつ）や `||`（または）で複数の条件を組み合わせることができる。
- switch文で `case` / `break` / `default` を正しく書き、値ごとの処理を実装できる。
- 条件分岐が複雑なときに `if / else if` と `switch` を使い分けられる。

---

**つまずき・気づき**
- `==` は型変換されるため `12 == "12"` が `true` になる。意図しない挙動を防ぐため原則 `===` を使う。
- `10 < x < 30` という数学的な書き方はJSでは使えない。`x > 10 && x < 30` のように `&&` で分けて書く。
- switch文で `break;` を書き忘れると、条件に合ったcaseの次のcaseも実行され続けてしまう（フォールスルー）。必ず各caseの末尾に書く。
- switch文の `{ }` の後ろにセミコロンは不要（if文と同じ）。
