# 今日学んだこと
### 2026-06-17〜23｜第2章ラスト〜第3章 グループ分けとCSSの基本（2-13〜3-16）

**何をしたか**
- 第2章の最後として、ページ全体を意味のあるかたまりに分ける **グループ分け用タグ**（`header`・`nav`・`article`・`section`・`main`・`aside`・`footer`・`div`）の役割と使い分けを学びました。
- 第3章でCSSの基礎（CSSとは／適用する3つの方法／`style.css`の作成と読み込み／セレクター・プロパティ・値の基本文法・記述ルール・単位・子孫セレクター）を習得しました。
- 文字や文章を装飾する **`font-size`・`font-family`・`font-weight`・`line-height`・`text-align`** の各プロパティと、フォントの種類・ジャンプ率・行間などのデザイン上の考え方を学びました。
- **Webフォント**の仕組みと、Google Fontsを使ってデバイスにフォントが無くても表示させる実装手順を学びました。
- **色の指定方法**（カラーコード／RGB値・rgba／色の名前）と、`color`・`background-color` プロパティ、配色の基礎（色相・明度・彩度、暖色／寒色、ベース・メイン・アクセントカラーの比率）を学びました。
- **背景まわりのプロパティ**（`background-image`・`background-repeat`・`background-size`・`background-position`・一括指定の`background`）を学びました。
- 要素の **幅・高さ**（`width`・`height`）と各種単位（`px`・`%`・`em`・`rem`・`vw`・`vh`）、**余白**（`margin`・`padding`・`border`）のボックスの概念を学びました。
- `margin`・`padding` の **四辺一括指定**（上右下左＝時計回り）、CSSのコメントアウト（`/* */`）、余白で関連情報をグループ化するデザインの考え方を学びました。
- 要素のまわりに線を引く **`border` 関連プロパティ**（`border-width`・`border-style`・`border-color`・一括指定の`border`）を学びました。
- **リストの装飾**（`list-style-type`・`list-style-position`・`list-style-image`・一括指定の`list-style`）を学びました。
- **クラスとID**を使った部分的なスタイル指定（`.クラス名`／`#ID名`、タグ名との併用、優先順位、命名ルール）を学びました。
- **Flexbox**でレイアウトを組む方法（`display: flex`・`flex-direction`・`flex-wrap`・`justify-content`）を学びました。

**学んだ用語**

| 用語 | やさしい説明 |
|---|---|
| グループ分け | 関連する要素どうしをタグで囲み、ひとかたまりにまとめること。レイアウトを整える土台になる |
| `<header>` | ページ上部のかたまりを囲むタグ。ロゴ・ページタイトル・ナビなどが入る。`<head>` 要素とは別物なので注意 |
| `<nav>` | 「navigation」の略。メインのナビゲーションメニューを囲むタグ。`<header>` の中に入ることが多い |
| `<article>` | 「記事」の意味。その部分だけを切り出しても独立して成り立つ内容（ニュース記事・ブログ記事など）に使う |
| `<section>` | 1つのテーマを持つ意味のあるかたまりを囲むタグ。`<article>` と違い、それ単体では完結しない |
| `<main>` | そのページの核となるメインコンテンツ全体を囲むタグ。中に複数のグループが入ることも多い |
| `<aside>` | 本文ではない補足情報（サイドバーなど）を囲むタグ。メインとの関連性が低い情報に使う |
| `<footer>` | ページ下部のかたまりを囲むタグ。コピーライトやSNSリンクなどが入る |
| `<div>` | 特に意味を持たないタグ。デザイン目的でまとめたい時など、他に適切なタグがない場合に使う |
| CSS | 「Cascading Style Sheets」の略。HTMLの見た目（色・サイズ・レイアウトなど）を装飾するための言語。拡張子は `.css` |
| `<link>` | `<head>` 内に書き、外部の別ファイル（CSSなど）を読み込むタグ |
| `rel` 属性 | `<link>` に付ける属性。読み込むファイルの種類（関係性）を指定する。CSSなら `stylesheet` |
| `href` 属性 | `<link>` に付ける属性。読み込むCSSファイルの場所（ファイル名）を指定する |
| `<style>` タグ | `<head>` 内に書き、その中に直接CSSを記述するタグ。そのHTMLファイル内だけに適用される |
| `style` 属性 | HTMLタグの中に直接CSSを書く属性。書いたタグだけに適用され、優先順位が最も高い |
| `@charset "UTF-8";` | CSSファイルの1行目に書く記述。文字化けを防ぐ。これより前にコードを書くとエラーになる |
| セレクター | 「どの部分を装飾するか」を指定する部分。タグ名・クラス・IDなどを書く |
| プロパティ | セレクターで指定した部分の「何を変えるか」を書く部分（文字色・文字サイズなど） |
| 値 | 「見た目をどのように変えるか」を書く部分（赤色にする・20pxにするなど） |
| `{ }`（波括弧） | セレクターに続けて、プロパティと値のまとまりを囲む記号 |
| `:`（コロン） | プロパティと値の間に書いて区切る記号 |
| `;`（セミコロン） | 値の最後に付けて、プロパティどうしを区切る記号 |
| 絶対値／絶対単位 | `px` のように、表示領域や親要素のサイズに影響されず、指定したサイズのまま表示される値・単位 |
| 相対値／相対単位 | `%`・`em`・`rem`・`vw`・`vh` のように、基準となる要素や表示領域のサイズによって変動する値・単位 |
| 子孫セレクター | セレクターを半角スペースで区切り、「ある要素の中にある要素」だけに装飾を指定する書き方（例：`div p`） |
| `font-size` | 文字の大きさを指定するプロパティ。値は `px`・`rem`・`%` やキーワードで指定する |
| `font-family` | フォント（書体）の種類を指定するプロパティ。複数をカンマ区切りで指定でき、先頭から順に適用される |
| ジャンプ率 | 見出しと本文の文字サイズの比率のこと。高いと躍動的、低いと上品で落ち着いた印象になる |
| 明朝体（serif系） | 縦画が太く横画が細い、「ウロコ」のある書体。丁寧で和風な印象。長文に適する |
| ゴシック体（sans-serif系） | 縦横の太さがほぼ同じで装飾の少ない書体。読みやすく、見出しや短文に向く |
| 装飾系フォント | 見せることを目的とした書体。長文には不向きで、見出しや短文にとどめて使う |
| `font-weight` | 文字の太さを指定するプロパティ。`normal`・`bold` などのキーワードや 1〜1000 の数値で指定する |
| `line-height` | 行の高さ（行間）を指定するプロパティ。単位なしの数値で指定するのが好まれる。おすすめは 1.5〜1.9 |
| `text-align` | テキストを揃える位置を指定するプロパティ。`left`（左）・`right`（右）・`center`（中央）・`justify`（両端）から指定する |
| Webフォント | フォントファイルをWebサーバー上に置くことで、ユーザーがインストールしていなくても表示できる技術 |
| Google Fonts | Googleが提供する無料のWebフォントサービス。導入が簡単で手軽に使える |
| `preconnect` | Webフォントなど外部サーバーへ事前に接続準備しておくための `<link>` の指定。表示を速くする |
| カラーコード | `#`（ハッシュ）で始まる3桁または6桁の英数字で色を表す方法。左2桁=赤・中2桁=緑・右2桁=青の度合い。`#ffffff`=白、`#000000`=黒。同じ数値が並ぶ場合は3桁に省略可（例：`#ff0000`→`#f00`） |
| RGB値 | `rgb(赤, 緑, 青)` の形式で色を表す方法。各値は0〜255で、0が最も暗い。`rgb(255,255,255)`=白 |
| Alpha値（rgba） | `rgba(赤, 緑, 青, 透明度)` の形式で透明度も指定する書き方。透明度は0〜1で、0=透明、1=不透明 |
| 無彩色 | 白・グレイ・黒など色味のない色。カラーコードでは赤・緑・青の3組が同じ値（例：`#9a9a9a`）になる |
| 色相 | 赤・黄・緑・青などの「色みの違い」のこと。円状に並べたものを色相環という |
| 補色／類似色相 | 色相環で反対にある色が補色、隣り合う色が類似色相 |
| 明度 | 色の明るさの度合い。高いと白に近づき爽快、低いと黒に近づき落ち着いた印象になる |
| 彩度 | 色の鮮やかさの度合い。高いと華やか、低いと渋く大人っぽい印象になる |
| 暖色／寒色／中性色 | 赤中心で暖かい印象が暖色、青中心で涼しい印象が寒色、緑・紫など温度を感じない色が中性色 |
| ベースカラー | デザインの基盤となる背景色などに使う色。比率の目安は約70% |
| メインカラー | デザインで最も見せたいテーマカラー。比率の目安は約25% |
| アクセントカラー | ボタンなど目立たせたい部分に使うワンポイントの色。比率の目安は約5% |
| `background-image` | 要素の背景に画像を設置するプロパティ。`url(画像のパス)` で指定する。`none` で背景画像なし |
| `background-repeat` | 背景画像の繰り返し方を指定するプロパティ。`repeat`／`repeat-x`（横）／`repeat-y`（縦）／`no-repeat` |
| `background-size` | 背景画像の大きさを指定するプロパティ。`cover`（領域を埋める・はみ出しは切れる）／`contain`（全体を表示・余白ができる）／数値 |
| `background-position` | 背景画像の表示開始位置を指定するプロパティ。「横方向 縦方向」の順。デフォルトは `left top` |
| `background`（一括指定） | 背景関連の値（色・画像・繰り返し・位置・サイズなど）をまとめて指定するショートハンド。`size` は `position` の直後に `/` で区切る |
| `width` / `height` | 要素の幅・高さを指定するプロパティ。値は数値（`px`・`%` など）や `auto`。インライン要素（`<a>`・`<span>`）には効かない |
| `vw` / `vh` | ビューポート（ブラウザーの表示領域）の幅・高さを基準とした相対単位。`50vw`=表示幅の半分、`50vh`=表示高さの半分 |
| `em` | 親要素のフォントサイズを基準とした相対単位。親が16pxなら `1em` = 16px |
| `margin` | 要素の外側の余白（他の要素との距離）を指定するプロパティ。`-top`／`-bottom`／`-left`／`-right` で個別指定できる |
| `padding` | 要素の内側の余白（枠線から中身までの距離）を指定するプロパティ |
| `border` | margin と padding の間にある枠線のこと。`border-width`・`border-style`・`border-color` の総称でもある |
| 四辺の一括指定 | `margin`・`padding`・`border-width` などで半角スペース区切りにすると、値の数で適用範囲が変わる。1個=四辺すべて／2個=上下・左右／3個=上・左右・下／4個=上・右・下・左（時計回り） |
| CSSのコメントアウト | `/*` と `*/` で囲んだ部分。ブラウザーに反映されないメモ。複数行も書ける。一時的にCSSを無効化する用途にも使う |
| 近接（余白でグループ化） | 近くに置いたものを人は「関連している」と認識する性質。関連する情報は近づけ、関連しない情報は余白を空けて離すとレイアウトが伝わりやすくなる |
| `border-width` | 線の太さを指定するプロパティ。`thin`／`medium`／`thick` のキーワードか数値。`border-style` の初期値は `none` なので同時に指定が必要 |
| `border-style` | 線の種類を指定するプロパティ。`none`／`solid`（1本線）／`double`（2本線）／`dashed`（破線）／`dotted`（点線）／`groove`・`ridge`・`inset`・`outset`（立体的な線） |
| `border-color` | 線の色を指定するプロパティ。カラーコード・RGB値・色名で指定。指定しないと黒になる |
| `border`（一括指定） | `border-width`・`border-style`・`border-color` をまとめて指定するショートハンド（順不同）。`border-top`／`-bottom`／`-left`／`-right` で辺ごとに指定できる |
| `list-style-type` | リスト項目の先頭マーカーの種類を指定するプロパティ。`none`／`disc`（黒丸）／`circle`（白丸）／`square`（黒四角）／`decimal`（数字）／`hiragana`・`katakana`・`cjk-ideographic`（漢数字）など多数 |
| リストマーカー | リスト項目の先頭に表示されるマーク。未指定だと `<ul>` は黒丸（disc）、`<ol>` は数字（decimal）で表示される |
| `list-style-position` | マーカーをボックスの外側（`outside`）か内側（`inside`）に表示するか指定するプロパティ |
| `list-style-image` | リストマーカーに画像を使うプロパティ。`url(画像のパス)` で指定。`none` で画像なし |
| `list-style`（一括指定） | `list-style-type`・`-position`・`-image` をまとめて指定するショートハンド。type と image を両方指定すると image が優先される |
| クラス（class） | タグに付けられる属性の1つ。CSSでは `.クラス名` で指定する。**1ページ内で何度でも**使える。HTMLの `class` 属性値には `.` を付けない |
| ID（id） | タグに付けられる属性の1つ。CSSでは `#ID名` で指定する。**1ページ内で1度しか**使えない。HTMLの `id` 属性値には `#` を付けない |
| `タグ名.クラス名` / `タグ名#ID名` | タグ名にクラス・IDを続けて書くと、そのタグかつそのクラス・IDが付いた箇所だけに適用される |
| CSSの優先順位（class と id） | 同じタグにクラスとIDで違う装飾を指定した場合、**IDで指定した方が優先**される |
| Flexbox | 「Flexible Box Layout Module」の略。複雑なレイアウトも簡単に組める仕組み。現在のレイアウトの主流（以前は `float` が主流だった） |
| Flexコンテナー / Flexアイテム | Flexboxで、子要素を囲む親要素がFlexコンテナー、中に並ぶ子要素がFlexアイテム |
| `display: flex` | 親要素（Flexコンテナー）に指定すると、中の子要素が横並びになる。Flexboxの起点となる指定 |
| `flex-direction` | 子要素を並べる向きを指定するプロパティ。`row`（初期値・左→右）／`row-reverse`（右→左）／`column`（上→下）／`column-reverse`（下→上） |
| `flex-wrap` | 子要素を1行にするか折り返すか指定するプロパティ。`nowrap`（初期値・1行）／`wrap`（折り返し上→下）／`wrap-reverse`（折り返し下→上） |
| `justify-content` | 子要素の水平方向の揃え位置を指定するプロパティ。`flex-start`（初期値・左揃え）／`flex-end`（右揃え）／`center`（中央）／`space-between`（両端＋均等）／`space-around`（均等） |

```html
<!-- ===== 2-13 グループ分けをしよう ===== -->

<!-- ページ全体の構造をグループ分けする -->
<header>                            <!-- ページ上部（ロゴ・タイトル・ナビなど） -->
  <h1>ページタイトル</h1>
  <nav>                            <!-- メインのナビゲーションメニュー -->
    <ul>
      <li><a href="#">サービス紹介</a></li>
      <li><a href="#">料金</a></li>
      <li><a href="#">お問い合わせ</a></li>
    </ul>
  </nav>
</header>

<main>                             <!-- ページの核となるメインコンテンツ全体 -->
  <article>                        <!-- 単体で独立して成り立つ記事部分 -->
    <h2>記事タイトル</h2>
    <p>最新のスマートフォン情報！新機種が発表されましたね！</p>
  </article>

  <section>                        <!-- 1つのテーマを持つかたまり -->
    <h2>その他のおすすめ記事</h2>
    <ul>
      <li><a href="#">スマートウォッチって使える?使えない?</a></li>
      <li><a href="#">ランニングにはこのガジェットがおすすめ！</a></li>
    </ul>
  </section>
</main>

<aside>                            <!-- 本文ではない補足情報（サイドバーなど） -->
  <h3>中の人ってこんな人</h3>
  <p>このWebサイトで情報を発信しているManaです。よろしくお願いします！</p>
</aside>

<footer>                           <!-- ページ下部（コピーライト・SNSリンクなど） -->
  <ul>
    <li><a href="#">Facebook</a></li>
    <li><a href="#">Twitter</a></li>
  </ul>
  <p>Copyright 2019 Mana</p>
</footer>

<!-- div：意味を持たず、デザイン目的などでまとめたい時に使う -->
<div>
  <img src="phone1.jpg" alt="スマートフォン画像">
  <p>画面を正面からみた様子</p>
</div>
```

```html
<!-- ===== 3-2 CSSを適用させる方法（3つ） ===== -->

<!-- 【方法1】外部CSSファイルを読み込む（最も一般的・おすすめ） -->
<!-- <head> 内に <link> タグを書く -->
<head>
  <meta charset="UTF-8">
  <title>猫の実態</title>
  <meta name="description" content="猫の好きなもの、日々の生活をご紹介">
  <link rel="stylesheet" href="style.css">  <!-- rel=種類, href=ファイル名 -->
</head>

<!-- 【方法2】<head> 内に <style> タグで書く（そのファイル内だけに適用） -->
<head>
  <style>
    h1 { color: #f00; }
    p { font-size: 18px; }
  </style>
</head>

<!-- 【方法3】HTMLタグの中に style 属性で書く（そのタグだけに適用・優先度最高） -->
<h1 style="color: #f00;">猫の一日</h1>
<p style="font-size: 18px;">ひたすら寝ています。</p>
<!-- ※ style 属性ではセレクターや { } の記述は不要 -->
```

```css
/* ===== 3-3 CSSファイルを作ろう（style.css） ===== */

@charset "UTF-8";              /* 必ず1行目に書く。文字化け防止 */

body {
  background-color: #fffeee;  /* ページ全体の背景色（淡い黄色） */
}

h1 {
  color: #0bd;                /* 見出しの文字色（水色） */
}

p {
  font-size: 20px;            /* 段落の文字サイズを20pxに指定 */
}
```

```css
/* ===== 3-4 CSSの基本の書き方 ===== */

/* 基本文法：セレクター { プロパティ: 値; } */
h1 {
  color: red;
}

/* 1行で書くこともできる */
h1 { color: red; }

/* 複数のセレクターに同じ装飾 → カンマ区切り */
h1, p { color: red; }

/* 1つのセレクターに複数の装飾 → セミコロンで区切る */
h1 { color: red; font-size: 20px; }

/* 子孫セレクター：半角スペースで区切ると「要素の中の要素」に指定できる */
/* 下記は <div> に囲まれた <p> だけ赤くなる（囲まれていない <p> は変化なし） */
div p {
  color: red;
}
```

```css
/* ===== 3-5 文字や文章を装飾しよう ===== */

/* font-size：文字の大きさ */
html { font-size: 100%; }      /* 基準サイズ（ブラウザー設定に追従する相対値） */
h1   { font-size: 2rem; }      /* 基準の2倍 */
h2   { font-size: 20px; }      /* 基準と無関係な絶対値 */

/* font-family：フォントの種類。カンマ区切りで先頭から順に適用 */
h1 {
  font-family: serif;          /* 明朝系のキーワード指定 */
}
h2 {
  /* 日本語フォント名やスペースを含む名前は " " で囲む。英語表記も併記する */
  font-family: "游ゴシック体", "Yu Gothic", YuGothic, sans-serif;
}
p {
  font-family: "ヒラギノ丸ゴ Pro W4", "Hiragino Maru Gothic Pro", sans-serif;
}

/* font-weight：文字の太さ */
p {
```

**つまずき・気づき**
- `<header>`（ページ上部のかたまり）と `<head>`（HTML冒頭の設定領域）は **まったく別のタグ**。名前が似ているので混同に注意。
- `<article>` と `<section>` の違い：`<article>` は「それ単体で独立して成り立つ記事」、`<section>` は「1つのテーマを持つが単体では完結しないかたまり」。迷ったら「切り出して独立するか」で判断する。
- `<div>` は意味を持たないタグなので、適切な専用タグ（`section` など）があるならそちらを優先し、最終手段として使う。
- CSSの適用は **外部CSSファイル（方法1）が最も一般的**。`<style>`タグや `style`属性はまとめて管理しづらいため、特別な理由がなければ方法1を使う。
- `@charset "UTF-8";` は **必ずCSSファイルの1行目**に書く。これより前に他のコードを書くとエラーになる。
- 値の最後の `;`（セミコロン）は最後の1つなら省略可能だが、後から追加した時にエラーになりやすいため、どの行でも必ず付ける癖をつけておくのが安全。
- 複数セレクターは `,`（カンマ）で区切る。`h1 p { ... }`（スペース区切り）は **子孫セレクター**になり意味が変わるので注意。
- 数値を指定する時は値が `0` の場合を除き **単位を必ず付ける**。`font-size: 20;`（単位なし）は無効。
- `font-family` で日本語名やスペースを含む名前は `" "` で囲む。ブラウザーによっては日本語名を識別できないため **英語表記も併記**し、最後に `sans-serif` などの総称を入れて指定フォントが無い環境に備える。
- フォントは **1デザインにつき1〜3種類まで**にとどめる。装飾系フォントや太字（`font-weight: bold`）は見出し・キーワードなど要所のみに使い、本文には使わない。
- `line-height` は **単位なしの数値**で指定するのがおすすめ。フォントサイズと連動するため、サイズ変更時も崩れにくい。行の高さが文字サイズより小さいと行が重なるので注意。
- `text-align: center`（中央揃え）は行の開始位置がバラバラになり長文では読みづらい。中央揃えは2〜3行程度の短文向き。長文は左揃え（`left`）や両端揃え（`justify`）にする。
- Webフォントは表示までに外部ファイルの読み込みが発生する。Google Fontsのコードには `preconnect` の `<link>` も含まれるので、**発行されたコードはまとめて `<head>` 内に貼り付ける**。
- カラーコードは先頭の `#`（ハッシュ）を忘れない。同じ数値が並ぶ6桁は3桁に省略できる（`#ffffff`→`#fff`、`#ff0000`→`#f00`）が、並んでいない場合は省略不可。
- 透明度を指定したい時は `rgb` ではなく `rgba` を使い、第4の値（透明度）を 0〜1 で書く（`0`=透明、`1`=不透明）。
- 無彩色（白・グレイ・黒）はカラーコードの **赤・緑・青の3組がすべて同じ値**になる（例：`#9a9a9a`、`#646464`）。
- 配色は色だけでなく **比率**も重要。ベース70%・メイン25%・アクセント5% が基本の目安。4色以上使う時はベースの割合は変えず、メインカラーを分割すると全体がまとまる。
- `background-size` の `cover` は領域いっぱいに広げるため画像の一部が切れることがあり、`contain` は全体を表示するが余白ができることがある。用途で使い分ける。
- `background-position` は「**横方向 縦方向**」の順で書く（デフォルトは `left top`）。
- 一括指定の `background` では、`background-size` の値は `background-position` の直後に **`/`（スラッシュ）で区切って**書く必要がある（例：`center bottom/cover`）。指定しなかったプロパティには初期値が適用される。
- `width`・`height` は **インライン要素（`<a>`・`<span>` など）には効かない**。サイズを変えたい時はブロック要素に指定する。
- ブロック要素（`<div>`・`<p>` など）の `width` は初期値が `auto` で、指定しないと親要素の幅いっぱい（親と同じ幅）に広がる。`width` を `%` で指定すると親要素の幅に対する比率になる。
- 余白の概念は「margin（外側）＞ border（枠線）＞ padding（内側）＞ 中身」の入れ子構造。margin は他の要素との距離、padding は枠線から中身までの距離、という違いを押さえる。
- `margin`・`padding`・`border-width` の四辺一括指定は **値の数で意味が変わる**（1個=四辺／2個=上下・左右／3個=上・左右・下／4個=上右下左の時計回り）。順番を間違えると意図しない辺に余白がつくので注意。
- 枠と文字の間（padding）は **文字サイズの1〜1.5倍程度**を目安にとると読みやすい（16pxなら20pxほど）。「ギリギリ収まればOK」ではなく、見やすい余白を確保する。
- 余白は「近接」のデザイン原則に使える。**関連する情報は近づけ、関連しない情報は離す**ことで、グループの区切りが言葉なしでも伝わる。
- CSSのコメントは HTMLの `<!-- -->` ではなく **`/* */`**。複数行も書け、一時的にCSSを無効化する用途にも使える。
- `border-style` の初期値は `none`（非表示）。`border-width` や `border-color` だけ指定しても **線は表示されない**ので、`border-style` を必ず一緒に指定する（または一括指定の `border` を使う）。
- `border-color` を指定しないと線は黒になる。線は文字色より薄い色を使うとコンテンツの邪魔にならずスッキリ見える。
- `list-style-type` は **`<ul>` または `<ol>` に指定**する（`<li>` ではない）。未指定だと `<ul>`=黒丸、`<ol>`=数字。
- `list-style-image` は1種類の画像しか指定できないため、基本は箇条書きリスト（`<ul>`）で使う。複雑な画像はコンテンツの邪魔になるので避ける。`list-style`（一括）で type と image を両方書くと **image が優先**される。
- HTMLの `class`／`id` 属性値には **ピリオドやハッシュを付けない**（`class="blue"`）。付けるのはCSS側のセレクター（`.blue`／`#orange`）だけ。
- **IDは1ページに1度しか使えない**（重複させるとCSSが正しく効かない）。何度も使い回す装飾はクラスにする。レイアウトの枠組みなど一意な部分にIDを使うとよい。
- 同じタグにクラスとIDの両方で違う装飾を指定すると、**IDが優先**される。
- クラス名・ID名のルール：①空白（スペース）を入れない ②英数字・ハイフン（`-`）・アンダースコア（`_`）で書く ③1文字目は必ず英字。日本語名はブラウザーによってエラーになる可能性があるため英数字で揃える。
- Flexboxは **親要素（Flexコンテナー）に `display: flex;` を指定**するのが起点。子要素ではなく親に書く点に注意。`flex-direction`・`flex-wrap`・`justify-content` も親側に指定する。
- `flex-direction` の初期値は `row`（左→右の横並び）。`flex-wrap` の初期値は `nowrap`（折り返さず1行）。`justify-content` の初期値は `flex-start`（左揃え）。
