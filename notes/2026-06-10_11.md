# 今日学んだこと
### 2026-06-10・11｜Webサイトの基本と制作の流れ

**何をしたか**
- Web制作を始める前の前提知識として、教材 第1章（1-1〜1-7）を通読しました。
- 「よいデザインとは何か」「Webサイトの種類」「ユーザビリティ（使いやすさ）」を整理しました。
- インターネット・Web・サーバー／クライアント・URL という **Webの仕組み** を理解しました。
- デバイスとブラウザーの種類、そして **制作の流れ6工程**（企画→サイトマップ→ワイヤーフレーム→デザイン→コーディング→公開）を確認しました。
- 1-7後半では、制作に必要な **HTML・CSS・画像ファイルの拡張子と命名ルール、標準フォルダ構成** という具体的な知識を学びました。

**学んだ用語**
| 用語 | やさしい説明 |
|---|---|
| デザイン | 「伝えるための手段」。見栄えだけでなく、相手に内容を伝え問題を解決すること |
| コーポレートサイト | 企業の公式サイト。会社概要・製品・採用情報などを載せる |
| プロモーションサイト（特設サイト） | 特定の商品・サービス・イベントを告知するサイト |
| ポートフォリオサイト | デザイナー等が自分の作品・実績を見せるサイト |
| ショッピングサイト（ECサイト／オンラインストアサイト） | ネットで商品を売るサイト |
| メディアサイト | ニュースや読み物記事で構成するサイト（ブログもこの一種） |
| SNS | ソーシャル・ネットワーキング・サービスの略。リアルタイム交流が得意 |
| ユーザビリティ | Webサイトの「使いやすさ」のこと |
| コントラスト | 色の明暗差。背景色と文字色に差があると読みやすい |
| グレースケール | 白黒のこと。配色のコントラスト確認に使う |
| インターネット | 世界中のコンピューターをつなぐ「ネットワーク（仕組み）」 |
| Web（World Wide Web） | インターネットの機能の1つ。情報を発信・閲覧する仕組み |
| Webサーバー | 情報を公開・保管するコンピューター |
| Webクライアント | 情報を受け取る側（私たちのPCやスマホ） |
| 要求（リクエスト）／応答（レスポンス） | クライアントが要求を送り、サーバーが応答してページが表示される流れ |
| URL | Webサイトの「住所」。各サイト固有のもの |
| プロトコル | URLの先頭部分（例：`http://`）。通信の方式を表す |
| ドメイン名 | URLの中の `example.com` のような部分 |
| ディレクトリ名（フォルダー名） | URL内のフォルダーを表す部分 |
| ファイル拡張子 | `.html` のように、ファイルの種類を表す末尾 |
| ハイパーリンク（リンク） | URLを入力せずページ間を移動できる仕組み |
| デバイス | 「装置」の意味。①端末 と ②周辺機器 の2つの意味がある |
| iOS / Android | スマホ等を動かすシステム（OS）。iOSはApple製、AndroidはGoogle製 |
| IoTデバイス | Internet of Things。身の回りのモノにネット機能を付けた端末 |
| デバイスドライバー | 周辺機器を使うために必要なソフトウェア |
| ブラウザー | サーバーから来たコードを解読し、Webページを見やすく表示するソフト |
| 標準ブラウザー | 制作時に表示確認の基準として決めておくブラウザー |
| メインゴール／サブゴール | サイトで達成したい主目標と、それを支える小目標 |
| ターゲットユーザー | 目標達成の核となるユーザー層（性別・年齢・職業など） |
| ペルソナ | ターゲットをさらに具体化した、実在の人物のような人物像 |
| サイトマップ | サイトの構成（ページとリンク関係）を図にしたもの |
| 階層 | 関連ページをグループ分けしてできる構造の深さ。なるべく2〜3階層以内に |
| ワイヤーフレーム | サイトの骨組みをラインとボックスで描いた設計図 |
| ファーストビュー | ページを開いた時、最初に画面に見える範囲 |
| デザインカンプ | 完成イメージを作り込んだデザインの見本 |
| コーディング | HTML/CSSなどでデザインを動く形にするファイル作成作業 |
| HTML | Webページのコンテンツ（文章・画像）を記述する言語。拡張子 `.html` |
| CSS | Webページの装飾（色・文字サイズ・配置）を記述する。拡張子 `.css` |
| JPG / PNG / GIF / SVG / WebP | Webで使える主な画像形式（写真→JPG、ロゴ/透過→PNG、簡単なアニメ→GIF、拡大に強い図形→SVG、軽量化→WebP） |

**できるようになったこと**: Web制作の全体像（よいデザインの考え方・サイトの6種類・ユーザビリティ・Webの仕組み・制作6工程）を自分の言葉で説明でき、HTML・CSS・画像ファイルの役割と拡張子を区別できる。
