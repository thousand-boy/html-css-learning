**学んだ用語**

| 用語 | やさしい説明 |
|---|---|
| `align-items` | Flexbox で、子要素を **縦方向（垂直方向）** のどの位置に置くかを決めるプロパティ。親要素に縦の余白がある時に効く |
| `stretch` | `align-items` / `align-content` の初期値。子要素を親の高さに合わせて引き伸ばして配置する |
| `flex-start` | 親要素の **開始位置（上）** に寄せて配置する値。上揃え |
| `flex-end` | 親要素の **終点（下）** に寄せて配置する値。下揃え |
| `center` | 中央に揃える値 |
| `baseline` | 文字の **ベースライン（下端の基準線）** で揃える値 |
| `100vh` | 高さの指定単位。`vh` は「viewport height」の略で、**画面（表示領域）の高さを基準**とした単位。`100vh` で画面の高さいっぱいになる |
| `align-content` | 子要素が **複数行に折り返した時** の、行全体の縦方向の揃えを決めるプロパティ。1行しかない（`nowrap`）と無効になる |
| `space-between` | 最初と最後の行（要素）を上下の端に置き、残りを均等な間隔で配置する値 |
| `space-around` | 上下端にある要素も含め、すべてを均等な間隔で配置する値 |
| CSSグリッド | 同じ大きさのボックスを格子状（タイル型）に並べるレイアウト手法。`display: grid` で始める |
| グリッドコンテナー | グリッド全体を囲む **親要素**。`display: grid` を指定する側 |
| グリッドアイテム | コンテナーの中に並ぶ **子要素**。実際にタイルとして配置されるボックス |
| グリッドギャップ | グリッドアイテム同士の間にできる **空白スペース** のこと |
| `display: grid` | その要素をグリッドコンテナーにするプロパティ値。これだけだと子要素は縦に並ぶ |
| `grid-template-columns` | 各列（横方向）のグリッドアイテムの **幅** を指定するプロパティ。半角スペース区切りで必要な列数だけ値を並べる |
| `grid-template-rows` | 各行（縦方向）のグリッドアイテムの **高さ** を指定するプロパティ。半角スペース区切りで行数だけ値を並べる |
| `gap` | グリッドアイテム同士の **余白** を指定するプロパティ。上下左右端は親コンテナーの枠に揃う。Flexbox でも使える |
| `fr` | CSSグリッドで使える単位。「fraction（比率）」の略で、幅を具体的な数値ではなく **割合** で指定する。画面幅に合わせて自動で伸縮する |
| Zの法則 | 視線が「左上→右上→左下→右下」と Z の字に動くという法則。初めて訪れる／画像の多いサイトで起きやすい |
| Fの法則 | 視線が左上から右、下へ下へと F の字に動くという法則。再訪／情報量の多いサイトで起きやすい |
| リセットCSS | ブラウザーが標準で当てている **デフォルトCSS** を打ち消し、どのブラウザーでも表示を統一するためのCSS |
| デフォルトCSS | ブラウザーがあらかじめ適用している標準のスタイル。ブラウザーごとに余白やフォントが微妙に異なる |
| `ress.css` | 本書で使う外部公開のリセットCSS。すべてを消すのではなく、デフォルトを活かしつつブラウザー間の最小限の誤差をなくすファイル |

**学んだ主な構文**

```css
/* ===== 3-16(続き) align-items：垂直方向の揃え ===== */

.container {
  display: flex;          /* Flexbox を有効化 */
  align-items: center;    /* 子要素を縦方向の中央に配置（値を変えると上下/ベースライン揃えに） */
  height: 100vh;          /* 画面の高さいっぱいに。100vh = 表示領域の高さ */
}
.item {
  background: #0bd;
  color: #fff;
  margin: 10px;
  padding: 10px;
}
/* align-items で使える値： stretch（初期値）/ flex-start / flex-end / center / baseline */
```

```css
/* ===== 3-16(続き) align-content：複数行にした時の揃え ===== */

.container {
  display: flex;
  flex-wrap: wrap;               /* 折り返しを許可（これがないと1行になり align-content は無効） */
  align-content: space-between;  /* 折り返した行を上下端に分けて均等配置 */
  height: 300px;                 /* 縦の余白を作るために高さを指定 */
}
.item {
  background: #0bd;
  color: #fff;
  margin: 10px;
  padding: 10px;
}
/* align-content で使える値： stretch（初期値）/ flex-start / flex-end / center / space-between / space-around */
```

```css
/* ===== 3-17 CSSグリッドの基本 ===== */

/* HTML 側は .container（親）の中に .item（子）を必要な数だけ並べる */

.container {
  display: grid;   /* グリッドコンテナーにする。この時点では子要素は縦に並ぶだけ */
}
.item {
  background: #0bd;
  color: #fff;
  padding: 10px;
}
```

```css
/* ===== 3-17 列幅と余白：grid-template-columns / gap ===== */

.container {
  display: grid;
  grid-template-columns: 200px 200px 200px; /* 1列に3つ、各200pxで横並び */
  gap: 10px;                                /* グリッドアイテム同士に10pxの余白 */
}
/* px 固定は画面が広いと右に余白ができ、狭いと見切れる */
```

```css
/* ===== 3-17 比率指定 fr と 行の高さ grid-template-rows ===== */

.container {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;  /* 1:1:1 の割合。画面幅に合わせて伸縮する */
  gap: 10px;
  grid-template-rows: 200px 200px;     /* 1行目・2行目とも高さ200px */
}
/* これで同じ大きさのボックスをタイル状に配置できる */
```

```html
<!-- ===== 3-18 リセットCSS（ress.css）の読み込ませ方 ===== -->

<head>
  <meta charset="utf-8">
  <title>WCB Cafe</title>

  <!-- CSS -->
  <!-- 【良い例】先に ress.css → 後に自作CSS の順で書く -->
  <link rel="stylesheet" href="https://unpkg.com/ress/dist/ress.min.css">
  <link href="css/style.css" rel="stylesheet">
</head>

<!-- ▲ ファイルをダウンロードせず、Web上のURLを直接書いても適用できる（要ネット接続） -->
```

```html
<!-- ===== 3-18 【悪い例】読み込み順が逆 ===== -->

<head>
  <meta charset="utf-8">
  <title>WCB Cafe</title>

  <!-- CSS -->
  <!-- 自作CSSが先、ress.css が後 → 自作CSSまでリセットされてしまう -->
  <link href="css/style.css" rel="stylesheet">
  <link rel="stylesheet" href="https://unpkg.com/ress/dist/ress.min.css">
</head>
```

**できるようになったこと**: `align-items` による Flexbox 子要素の垂直方向の揃え（上・下・中央・ベースライン）の指定。`flex-wrap: wrap` と `align-content` を組み合わせた複数行の揃え（`space-between`／`space-around` など）の指定。`display: grid` でのグリッドレイアウトの開始。`grid-template-columns`／`grid-template-rows` による列幅・行高の指定と、`gap` による余白付け。`fr`（比率）と `px`（固定）の使い分け。Z型・F型の視線の流れと、配置順・面積を踏まえた優先順位付け。`ress.css` を `<head>` 内で自作CSSより先に読み込ませる、ブラウザー差を消すリセットCSSの導入。

**つまずき・気づき**
- `align-content` は **複数行（`flex-wrap: wrap`）でないと効かない**。`flex-wrap: nowrap`（1行）のままだと無効になる点に注意。
- `align-items`（行内＝1行の縦揃え）と `align-content`（行同士＝複数行の縦揃え）は **役割が別物**。混同しやすい。
- `100vh` は画面（ビューポート）の高さ基準なので、`align-items: center` と併用すると要素を画面のちょうど中央に置ける。
- CSSグリッドは `display: grid` を付けただけだと **子要素が縦に並ぶだけ**。横に並べるには `grid-template-columns` で列幅を指定する必要がある。
- `grid-template-columns` / `grid-template-rows` は **半角スペース区切り** で、並べた値の数＝列数・行数になる（例：`200px 200px 200px` で3列）。
- `px` 固定は画面幅が広いと右に余白、狭いと見切れる。**画面幅に追従させたいなら `fr`** を使う。
- `gap` は CSSグリッド専用ではなく **Flexbox の子要素間にも使える**。
- リセットCSSは **読み込み順が命**。自作CSSの後に `ress.css` を書くと、後から読まれた `ress.css` が優先されて自作スタイルまで打ち消される。必ず `ress.css` を先、自作CSSを後に書く。
- `ress.css` は「すべてのデフォルトを消す」のではなく、**デフォルトを活かしつつブラウザー間の最小限の誤差をなくす**タイプのリセットCSS（同じ「リセット」でも方針はファイルにより異なる）。
- Web上のURLを直接 `link` で読み込む方式は、**ネット未接続だと適用されない**。オフライン作業ではファイルをダウンロードして読み込ませる必要がある。
- スライド注釈内に `warp` という表記があるが、正しいプロパティ値は `wrap`（折り返し）。注釈側の誤字で、CSSコード本体は `flex-wrap: wrap` と正しく書かれている。
